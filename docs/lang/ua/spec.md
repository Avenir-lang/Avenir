# Специфікація мови (коротка)

Цей документ дає короткий нормативний опис базового синтаксису та семантики
Avenir. Детальніші пояснення та приклади дивіться в інших файлах цього
каталогу.

## Лексична структура

- ідентифікатори: літери/`_`, далі літери/цифри/`_`
- ключові слова: `pckg`, `import`, `fun`, `struct`, `pub`, `mut`, `var`, `if`, `else`, `while`, `for`, `in`, `return`, `break`, `try`, `catch`, `throw`, `true`, `false`, `none`, `some`, `interface`
- літерали: `int`, `float`, `string`, `bytes`, `bool`, `none`, `some(...)`

## Типи

- примітиви: `int`, `float`, `string`, `bool`, `bytes`, `void`, `any`, `error`
- складені типи: `list<T>`, `dict<T>`, типи функцій `fun(...) | T`
- опціональні типи: `T?`
- об'єднані типи: `<T1|T2|...>`
- типи структур та інтерфейсів
- оголошення узагальнених типів у структур і функцій: `Name<T, U, ...>`
- використання узагальнених типів потребують явних аргументів типів

## Вирази

- арифметичні, порівняльні та логічні оператори
- конкатенація рядків через `+` дозволена лише для `string + string`
- індексація: `list[int]`, `bytes[int]`, `dict[string]`
- доступ до полів і методів: `expr.field`, `expr.method(...)`
- виклики узагальнених функцій: `fn<T, U>(...)`

## Оператори

- оголошення змінних: `var name | Type = expr;`
- присвоєння: `name = expr;`
- `if`, `while`, `for`, `for (item in list)`
- `return`, `break`, `throw`, `try/catch`

## Модулі

- кожен файл оголошує пакет через `pckg`
- імпорти використовують крапкові шляхи: `import std.net as net;`
- папкові модулі розв'язуються в `path/path.av` за правилом відповідності файл/структура

## Семантика виконання програми

- усі помилки виконання програми викидаються як значення `error` і можуть бути перехоплені через `try / catch`
- вбудовані функції та методи є частиною ядра мови й викликаються безпосередньо віртуальною машиною

Деталі дивіться в `syntax.md`, `types.md`, `operators.md`, `builtins.md` і `docs/std/`.
