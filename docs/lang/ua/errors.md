# Обробка помилок

Avenir використовує винятки для обробки помилок. Винятки представлені типом `error` і виникають у випадках:

- явного `throw`
- помилок часу виконання
- помилок вбудованих функцій

## Тип `error`

Тип `error` представляє виняткові ситуації:

```avenir
var e | error = error("щось пішло не так");
```

Помилки створюються через вбудовану функцію `error()`. Помилки часу виконання і помилки вбудованих функцій також автоматично створюються VM як значення `error`.

## Викидання винятків

Винятки викидаються через `throw`:

```avenir
throw error("щось пішло не так");
```

Вираз повинен мати тип `error`.

## Перехоплення винятків

Винятки перехоплюються через `try-catch`:

```avenir
try {
    throw error("сталася помилка");
} catch (e | error) {
    print(errorMessage(e));
}
```

Помилки часу виконання і помилки вбудованих функцій також можна перехоплювати:

```avenir
try {
    var n | int = toInt("abc");
    var x | int = 10 / 0;
    print(n + x);
} catch (e | error) {
    print(errorMessage(e));
}
```

Змінна в `catch` повинна мати тип `error`.

## Повідомлення про помилки

Текст помилки витягується за допомогою `errorMessage()`:

```avenir
var msg | string = errorMessage(e);
```

## Поширення винятків

Якщо виняток не перехоплено, він піднімається стеком викликів:

```avenir
fun f() | void {
    throw error("помилка в f");
}

fun g() | void {
    f();
}

fun main() | void {
    try {
        g();
    } catch (e | error) {
    }
}
```

Якщо виняток доходить до верхнього рівня і не перехоплюється, програма завершується з помилкою.

## Необроблені винятки

Необроблені винятки призводять до завершення програми:

```avenir
fun main() | void {
    throw error("необроблена помилка");
}
```

## Шаблони обробки помилок

### Проброс помилок

Функція може пробросити помилку далі, просто не перехоплюючи її:

```avenir
fun may_fail() | void {
    if (condition) {
        throw error("збій");
    }
}
```

### Обробка помилок

Функція може обробити помилку через `try-catch`:

```avenir
fun safe_operation() | void {
    try {
        may_fail();
    } catch (e | error) {
        print(errorMessage(e));
    }
}
```

### Повернення помилок

Зараз Avenir використовує насамперед винятки. Функції можуть повертати `error`, але винятки все одно викидаються через `throw` або виникають із помилок часу виконання та вбудованих функцій.

## Обмеження типу `error`

`error` — вбудований тип із повідомленням і розширюваними метаданими. Користувацькі типи помилок поки не підтримуються.

## Практичні рекомендації

- коректно обробляйте помилки
- робіть повідомлення зрозумілими
- використовуйте `try-catch` там, де справді можете обробити виняток
- документуйте функції, які можуть викидати помилки
