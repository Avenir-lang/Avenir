# Функції

Функції в Avenir є повноправними значеннями. Вони можуть бути іменованими
або анонімними (літералами функцій).

## Оголошення функції

Функції оголошуються за допомогою `fun`:

```avenir
fun name(param1 | type1, param2 | type2) | returnType {
    
}
```

Тип повернення обов'язковий. Для відсутності результату використовуйте `void`.

### Асинхронні функції

Функції можна оголошувати як асинхронні:

```avenir
async fun fetchNumber() | int {
    return 10;
}
```

Виклик async-функції повертає значення `Future`.


### `await`

`await` використовується всередині async-функцій для отримання результату:

```avenir
async fun main() | int {
    var a | int = await fetchNumber();
    return a + 1;
}
```

### Дженеріки

Функції можуть мати параметри типів:

```avenir
fun identity<T>(x | T) | T {
    return x;
}
```

Аргументи типів у виклику вказуються явно.

Виведення аргументів узагальнених типів поки не підтримується.

## Публічні функції

`pub fun` робить функцію доступною з інших модулів.

## Параметри

Параметри завжди мають явні анотації типів.

### Значення за замовчуванням

Параметри можуть мати значення за замовчуванням, відомі під час компіляції.

### Іменовані аргументи

Підтримуються іменовані аргументи. Позиційні аргументи мають іти раніше
іменованих.

## Повернення значень

Повернення виконується через `return`. Для `void` можна використовувати `return;` без значення.

## Літерали функцій і замикання

Анонімні функції підтримуються і можуть захоплювати змінні зовнішньої області
видимості.

## Типи функцій

Тип функції записується як:

```avenir
fun (int, int) | int
```

## Виклики функцій

Функції викликаються через круглі дужки, методи — через крапкову нотацію.

## `main`

Кожна програма повинна мати точку входу `main`.

```avenir
fun main() | void {
}
```

`main` також може бути асинхронною.
