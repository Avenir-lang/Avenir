# Функції

Функції в Avenir є повноправними значеннями. Вони можуть бути іменованими або анонімними (літералами функцій).

## Оголошення функції

Функції оголошуються за допомогою ключового слова `fun`:

```avenir
fun name(param1 | type1, param2 | type2) | returnType {
    // тіло
}
```

Тип повернення обов'язковий. Для відсутності результату використовуйте `void`.

### Асинхронні функції

Функції можна оголошувати як асинхронні за допомогою ключового слова `async`:

```avenir
async fun fetchNumber() | int {
    return 10;
}
```

Виклик async-функції автоматично породжує конкурентну задачу і повертає `Future<T>`. Викликач може зберегти future або одразу виконати `await`.

### Вирази await

Використовуйте `await` всередині async-функцій для отримання розв'язаного значення:

```avenir
async fun main() | int {
    var a | int = await fetchNumber();
    return a + 1;
}
```

`await` призупиняє поточну задачу до розв'язання future. Інші задачі можуть виконуватися, поки поточна призупинена.

### Конкурентний запуск

Виклик кількох async-функцій перед очікуванням забезпечує справжню конкурентність:

```avenir
async fun compute(x | int) | int {
    await asyncSleep(50000000);
    return x * 2;
}

async fun main() | int {
    var a | Future<int> = compute(10);
    var b | Future<int> = compute(20);
    var ra | int = await a;
    var rb | int = await b;
    return ra + rb;
}
```

Обидва виклики `compute` працюють конкурентно. Загальний час ~50мс, а не ~100мс.

### Узагальнені функції

Функції можуть оголошувати параметри типів після імені:

```avenir
fun identity<T>(x | T) | T {
    return x;
}

fun pickFirst<T, U>(a | T, b | U) | T {
    return a;
}
```

При виклику аргументи типів вказуються явно:

```avenir
var n | int = identity<int>(42);
var s | string = identity<string>("hello");
```

Виведення аргументів типів поки не підтримується.

Приклад:

```avenir
fun add(x | int, y | int) | int {
    return x + y;
}

fun print_hello() | void {
    print("Hello");
}
```

## Декоратори

Декоратори дозволяють обгортати функції додатковою поведінкою за допомогою синтаксису `@<вираз>`:

```avenir
@log
fun add(a | int, b | int) | int {
    return a + b;
}
```

Декоратор — це будь-який вираз, що обчислюється до функції, яка приймає функцію як аргумент і повертає функцію того самого типу. Наведене вище еквівалентне `add = log(add)`.

Декоратори застосовуються **один раз при ініціалізації модуля**, а не при кожному виклику функції. Це означає, що декоровані функції мають **нульовий runtime-оверхед** — оригінальний слот функції назавжди замінюється декорованою версією до виконання `main`.

### Вимоги до декораторів

Функція-декоратор повинна:
- Приймати рівно один параметр: функцію, яку декоруємо
- Повертати функцію з такою самою сигнатурою

```avenir
fun doubler(f | fun(int, int) | int) | fun(int, int) | int {
    return fun(a | int, b | int) | int {
        return f(a, b) * 2;
    };
}

@doubler
fun add(a | int, b | int) | int {
    return a + b;
}

// add(3, 4) тепер повертає 14 замість 7
```

### Параметризовані декоратори

Декоратори можуть приймати аргументи в дужках. Параметризований декоратор — це функція, що повертає декоратор:

```avenir
@cache(60)
fun compute(x | int) | int {
    return x * x;
}
```

Тут `cache(60)` викликається першим, а повернена функція використовується як декоратор.

### Декоратори та методи

Декоратори працюють як з методами екземпляра, так і зі статичними методами.

Для методів екземпляра тип методу включає receiver як перший параметр. Тому декоратор повинен приймати і повертати повний тип функції:

```avenir
struct Point {
    x | int
    y | int
}

fun method_doubler(f | fun(Point, int, int) | int) | fun(Point, int, int) | int {
    return fun(self | Point, dx | int, dy | int) | int {
        return f(self, dx, dy) * 2;
    };
}

@method_doubler
fun (self | Point).move_score(dx | int, dy | int) | int {
    return self.x + self.y + dx + dy;
}

// Статичний метод з декоратором
@validate
fun Point.origin() | Point {
    return Point{x = 0, y = 0};
}
```

### Статичні методи як декоратори

Статичні методи можна використовувати як декоратори, оскільки вони доступні статично:

```avenir
struct Logger {
}

fun Logger.log(f | fun() | void) | fun() | void {
    return fun() | void {
        print("Starting function");
        f();
        print("Function completed");
    };
}

fun Logger.validate(f | fun(int) | int) | fun(int) | int {
    return fun(x | int) | int {
        if x < 0 {
            print("Warning: negative input");
        }
        return f(x);
    };
}

// Використання статичних методів як декораторів
@Logger.log
fun test() | void {
    print("Hello, World!");
}

@Logger.validate
fun process(x | int) | int {
    return x * 2;
}
```

**Зверніть увагу:** Оскільки декоратори — це довільні вирази, будь-який вираз, що обчислюється до функції-декоратора, може бути використаний, включно з доступом до статичних методів як `Logger.log`. Методи екземпляра не можуть бути декораторами, оскільки потребують конкретного екземпляра.

**Зверніть увагу:** Декоровані методи оголошуються на верхньому рівні з синтаксисом receiver-а (`fun (self | T).name(...) ...`). Вони не оголошуються всередині блоку полів `struct { ... }`.

### Множинні декоратори

Декоратори можна комбінувати. Вони застосовуються знизу вгору (внутрішній першим):

```avenir
@dec1
@dec2
fun f(x | int) | int {
    return x;
}
// Еквівалентно: f = dec1(dec2(f))
```

### Узагальнені декоратори

Декоратори можуть бути узагальненими функціями з варіативними параметрами типів для універсального застосування:

```avenir
fun wrap<R, ...Args>(f | fun(Args...) | R) | fun(Args...) | R {
    return f;
}
```

При застосуванні до функції аргументи типів автоматично виводяться із сигнатури декорованої функції.

Використання:

```avenir
@wrap
fun add(a | int, b | int) | int {
    return a + b;
}

@wrap
fun greet(name | string, times | int) | string {
    return "hello";
}
```

У цих прикладах `wrap` інстанціюється з різними `(R, Args...)` на основі кожної декорованої функції.

## Варіативні дженеріки

Варіативні параметри типів дозволяють функціям приймати змінну кількість аргументів типів.

### Варіативні параметри типів

Оголошуйте варіативний параметр типу з `...` перед іменем:

```avenir
fun wrap<R, ...Args>(f | fun(Args...) | R) | fun(Args...) | R {
    return f;
}
```

- `...Args` оголошує `Args` як варіативний параметр типу (пакет типів)
- `Args...` розгортає пакет типів у позиції типу

### Розгортання пакету типів

Використовуйте `Name...` у позиціях типів для розгортання пакету типів:

```avenir
fun(Args...) | R    // функція, що приймає розгорнуті типи і повертає R
```

Коли `wrap` застосовується до `fun(int, string) | bool`, type checker виводить:
- `R = bool`
- `Args = (int, string)`

І `fun(Args...) | R` розгортається у `fun(int, string) | bool`.

## Публічні функції

Функції можна позначати як публічні за допомогою ключового слова `pub`:

```avenir
pub fun exported_function() | void {
    // Ця функція доступна з інших модулів
}
```

Приватні функції (без `pub`) доступні лише в межах того самого модуля.

## Параметри

Параметри функції повинні мати явні анотації типів:

```avenir
fun greet(name | string, age | int) | void {
    print(name);
    print(age);
}
```

### Значення за замовчуванням

Параметри можуть мати значення за замовчуванням:

```avenir
fun greet(name | string, greeting | string = "Hello") | void {
    print(greeting);
    print(name);
}
```

Значення за замовчуванням мають бути константами часу компіляції.

При виклику функції з параметрами за замовчуванням аргументи можна опустити:

```avenir
greet("Alice");              // greeting = "Hello" за замовчуванням
greet("Bob", "Hi");          // greeting = "Hi"
```

### Іменовані аргументи

Виклики функцій підтримують іменовані аргументи:

```avenir
fun create_point(x | int, y | int) | Point {
    return Point{x = x, y = y};
}

create_point(x = 10, y = 20);
create_point(y = 20, x = 10);  // Порядок не має значення
```

Іменовані аргументи можна поєднувати з позиційними, але позиційні мають іти першими:

```avenir
create_point(10, y = 20);      // OK
create_point(x = 10, 20);      // Помилка: позиційний після іменованого
```

## Повернення значень

Функції повертають значення за допомогою оператора `return`:

```avenir
fun add(x | int, y | int) | int {
    return x + y;
}
```

Функції з типом повернення `void` можуть використовувати `return` без значення:

```avenir
fun early_exit() | void {
    if (condition) {
        return;
    }
    // ...
}
```

## Літерали функцій

Літерали функцій (анонімні функції) підтримуються:

```avenir
var add | fun (int, int) | int = fun (x | int, y | int) | int {
    return x + y;
};
```

Літерали функцій можна передавати як аргументи і зберігати в змінних.

## Замикання

Літерали функцій можуть захоплювати змінні із зовнішньої області видимості (замикання):

```avenir
fun make_counter() | fun () | int {
    var count | int = 0;
    return fun () | int {
        count = count + 1;
        return count;
    };
}
```

Замикання захоплюють змінні за посиланням, що дозволяє модифікувати захоплені змінні.

## Типи функцій

Типи функцій задаються за допомогою ключового слова `fun`:

```avenir
var fn | fun (int, int) | int = add;
```

Типи функцій використовують структурну типізацію: два типи функцій рівні, якщо мають однакові типи параметрів і тип повернення.

## Виклики функцій

Функції викликаються через круглі дужки:

```avenir
var result | int = add(10, 20);
```

Узагальнені виклики включають аргументи типів:

```avenir
var value | int = identity<int>(10);
```

Методи викликаються через крапкову нотацію:

```avenir
var length | int = str.length();
```

## Функція main

Кожна програма повинна мати функцію `main`, що є точкою входу:

```avenir
fun main() | void {
    // Точка входу програми
}
```

Функція `main` викликається автоматично при запуску програми.

`main` також може бути асинхронною:

```avenir
async fun main() | int {
    var a | int = await fetchNumber();
    return a;
}
```
