# Словники (`dict`)

`dict` — вбудований узагальнений тип словника в Avenir з повною формою
`dict<K, V>`, де `K` — тип ключа, а `V` — тип значення.

Скорочена форма `dict<V>` еквівалентна `dict<string, V>` (ключі — рядки за
замовчуванням).

## Синтаксис

Літерали словника використовують фігурні дужки та пари `key: value`:

```avenir
var user | dict<any> = {
    name: "Alex",
    "age": 30,
    isAdmin: true,
};
```

Ключами можуть бути:

- ідентифікатори (`name`, `isAdmin`)
- рядкові літерали (`"age"`, `'role'`)

Завершальна кома необов'язкова.

## Типи

Тип словника може бути записаний двома способами:

**Коротка форма** — `dict<V>` (тип ключа за замовчуванням `string`):

```avenir
var scores | dict<int> = { alice: 10, bob: 12 };
var meta   | dict<any> = { env: "dev", retries: 3 };
```

**Повна форма** — `dict<K, V>` (явні типи ключа та значення):

```avenir
var scores | dict<string, int> = { math: 95, science: 88 };
```

Усі значення літерала повинні бути сумісні з `V`. Змішані типи значень
потребують явного об'єднаного типу:

```avenir
var mixed | dict< <int|string> > = { a: 1, b: "two" };
```

## Доступ

Крапковий доступ читає ключ:

```avenir
print(user.name);
```

Індексний доступ читає ключ динамічно:

```avenir
print(user["age"]);
```

Якщо при індексному доступі ключ відсутній, виникає помилка часу виконання. Якщо ключ
може бути відсутнім, використовуйте `dict.get()`.

## Вбудовані методи

Для `dict<K, V>`:

| Метод | Параметри | Повертає | Примітки |
| --- | --- | --- | --- |
| `length()` | — | `int` | кількість записів |
| `keys()` | — | `list<K>` | порядок не гарантовано |
| `values()` | — | `list<V>` | порядок не гарантовано |
| `has(key)` | `K` | `bool` | перевірка наявності |
| `get(key)` | `K` | `V?` | `none`, якщо відсутній |
| `set(key, value)` | `K`, `V` | `void` | змінює словник на місці |
| `remove(key)` | `K` | `bool` | повертає, чи існував ключ |

## Словники і структури

У структур фіксована схема та іменовані поля; словники динамічні й
індексуються рядками. Використовуйте структури для фіксованої форми даних, а
словники — для динамічних даних.

## Примітки

- Словники засновані на хеш-таблиці; порядок ітерації не гарантовано.
- `dict.set` змінює словник на місці.
- Наразі підтримуються лише ключі типу `string`. Параметр типу `K` дозволяє
  розширити набір допустимих типів ключів у майбутньому без зміни синтаксису.
