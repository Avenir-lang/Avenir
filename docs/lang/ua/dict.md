# Словники (`dict`)

`dict` — вбудований тип словника в Avenir. Ключі завжди мають тип `string`, а
значення статично типізовані.

## Синтаксис

Літерали словника використовують фігурні дужки та пари `key: value`:

```avenir
var user | dict<any> = {
    name: "Alex",
    "age": 30,
    isAdmin: true,
};
```

Ключами можуть бути:

- ідентифікатори (`name`, `isAdmin`)
- рядкові літерали (`"age"`, `'role'`)

Завершальна кома необов'язкова.

## Типи

Тип словника записується як `dict<T>`, де `T` — тип значення:

```avenir
var scores | dict<int> = { alice: 10, bob: 12 };
var meta   | dict<any> = { env: "dev", retries: 3 };
```

Усі значення літерала повинні бути сумісні з `T`. Змішані типи значень
потребують явного об'єднаного типу:

```avenir
var mixed | dict< <int|string> > = { a: 1, b: "two" };
```

## Доступ

Крапковий доступ читає ключ:

```avenir
print(user.name);
```

Індексний доступ читає ключ динамічно:

```avenir
print(user["age"]);
```

Якщо при індексному доступі ключ відсутній, виникає помилка часу виконання. Якщо ключ
може бути відсутнім, використовуйте `dict.get()`.

## Вбудовані методи

| Метод | Параметри | Повертає | Примітки |
| --- | --- | --- | --- |
| `length()` | — | `int` | кількість записів |
| `keys()` | — | `list<string>` | порядок не гарантовано |
| `values()` | — | `list<T>` | порядок не гарантовано |
| `has(key)` | `string` | `bool` | перевірка наявності |
| `get(key)` | `string` | `T?` | `none`, якщо відсутній |
| `set(key, value)` | `string`, `T` | `void` | змінює словник на місці |
| `remove(key)` | `string` | `bool` | повертає, чи існував ключ |

## Словники і структури

У структур фіксована схема та іменовані поля; словники динамічні й
індексуються рядками. Використовуйте структури для фіксованої форми даних, а
словники — для динамічних даних.

## Примітки

- Словники засновані на хеш-таблиці; порядок ітерації не гарантовано.
- `dict.set` змінює словник на місці.
