# Модулі

Avenir використовує модульну систему для організації коду. Кожен файл — це модуль, що визначається оголошенням `pckg`.

## Оголошення пакета

Кожен файл має починатися з:

```avenir
pckg <name>;
```

Ім'я пакета може бути простим ідентифікатором або dotted path:

```avenir
pckg main;
pckg std.net;
pckg app.utils;
```

Ім'я пакета визначає модуль і використовується під час імпорту.

## Імпорти

Модулі імпортуються через `import`:

```avenir
import <module_path>;
```

Шлях імпорту — це dotted identifier, що збігається з ім'ям пакета імпортованого модуля:

```avenir
import std.net;
import std.json as json;
```

### Аліаси імпортів

Імпорт може мати аліас:

```avenir
import std.json as json;
```

Якщо аліас не задано, локальним ім'ям стає останній сегмент шляху.

### Використання імпортованих модулів

Доступ до імпортованих символів здійснюється через крапкову нотацію:

```avenir
import std.net as net;

var sock | net.Socket = net.connect("example.com", 80);
```

## Відповідність файла і структури

**Файл можна імпортувати лише якщо він містить структуру з тим самим ім'ям, що й файл.**

### Правило

Якщо файл містить структури, **хоча б одна структура повинна мати ім'я файла** без розширення `.av`.

Приклади.

**Коректно:**

```
Файл: Point.av
Містить: struct Point { x | int; y | int; }
→ Можна імпортувати
```

**Некоректно:**

```
Файл: Point.av
Містить: struct Rectangle { x | int; y | int; }
→ Помилка компіляції: "file 'Point.av' does not contain struct 'Point'"
```

Файли без структур можна імпортувати вільно: правило застосовується лише до файлів, де структури є.

Якщо у файлі кілька структур, хоча б одна з них повинна збігатися з іменем файла.

## Імпорти через папки

Якщо папка `A` містить файл `A.av`, імпорт вказує лише на папку:

```
Папка: module/geometry
Файл: module/geometry/geometry.av
Імпорт: import module.geometry;
```

Компілятор автоматично резолвить `import module.geometry` у `module/geometry/geometry.av`.

### Порядок резолву

Для шляху на кшталт `module.A` компілятор намагається:

1. папковий варіант: `module/A/A.av`
2. плоский файл: `module/A.av`

Це правило діє і для `std.*`, і для звичайних модулів застосунку.

### Вкладені модулі

Для вкладених шляхів застосовується та сама логіка:

```
Папка: app/utils/
Файл: app/utils/utils.av
Пакет: pckg app.utils;
Імпорт: import app.utils;
```

## Резолв модулів

Компілятор резолвить імпорти так:

1. Для модулів `std.*`:
   - спочатку перевіряє `std/<path>/<last-segment>.av`
   - потім перевіряє `std/<path>.av`
   - за потреби використовує корінь репозиторію для каталогу `std/`

2. Для інших модулів:
   - спочатку перевіряє `<project-root>/<path>/<last-segment>.av`
   - потім перевіряє `<project-root>/<path>.av`

Корінь проєкту визначається як директорія entry-файла.

## Багатофайлові модулі

При папковому імпорті модуль може складатися з кількох файлів. Усі `.av` файли директорії об'єднуються в один модуль.

Приклад структури:

```
std/net/
  net.av
  socket.av
  server.av
```

Якщо файл містить структури, він повинен містити структуру, що збігається з ім'ям файла. Для допоміжних файлів можна додати приватну структуру-заглушку.

## Перевірки на етапі компіляції

Компілятор суворо перевіряє:

- відсутність файла або папки для імпорту
- невідповідність імені структури і файла
- існування папки без очікуваного файла

### Повідомлення про помилки

Компілятор намагається видавати зрозумілі повідомлення, наприклад:

```
Error: folder "module/geometry" exists but does not contain required file "geometry.av"
Expected: module/geometry/geometry.av
```

## Доступ до модуля

Після імпорту доступні публічні функції, структури та інтерфейси модуля:

```avenir
import std.net as net;

fun main() | void {
    var sock | net.Socket = net.connect("example.com", 80);
    sock.close();
}
```

## Видимість модулів

З інших модулів доступні лише `pub` функції, структури та інтерфейси.

## Примітки щодо аліасів імпорту

Якщо аліас не задано, локальним ім'ям стає останній сегмент шляху. Для зручності `std.http.client` і `std.http.server` можна імпортувати з `as http`.

## Приклади

```
Файл: utils.av
Пакет: pckg utils;
Імпорт: import utils;
```

### Папковий імпорт

```
Папка: math/
Файл: math/math.av
Пакет: pckg math;
Імпорт: import math;
```

## Структура модуля

Модуль може містити:

1. оголошення пакета
2. імпорти
3. структури
4. інтерфейси
5. функції

## Цикли імпортів

Цикли імпортів виявляються і повертаються як помилки. Модуль не може імпортувати сам себе прямо або опосередковано.
