# Середовище виконання

Програми Avenir виконуються на віртуальній машині (VM). VM стекова і
виконує байткод.

## Модель виконання

Програма проходить етапи:

1. парсинг вихідного коду
2. перевірка типів
3. компіляція у байткод (IR)
4. виконання у VM

## Віртуальна машина

VM використовує стек операндів і стек викликів.

### Стек

Стек зберігає:

- операнди виразів
- аргументи та результати викликів функцій
- локальні змінні

### Виклики функцій

Під час виклику VM:

1. кладе аргументи у стек
2. створює новий фрейм
3. виконує тіло функції
4. знімає результат
5. відновлює попередній фрейм

### Асинхронність: `async` і `await`

Avenir підтримує асинхронне виконання через `async` і `await`.

- виклик async-функції повертає значення `Future`
- `await` отримує результат `Future`
- середовище виконання використовує однопотоковий планувальник і цикл подій
- якщо значення `Future` ще не готове, поточна задача призупиняється і відновлюється пізніше

### Замикання

Замикання захоплюють змінні із зовнішньої області видимості. Захоплені
значення зберігаються в масиві захоплених значень.

## Вбудовані функції та методи

Вбудовані функції реалізовані на Go і викликаються напряму з VM.

- функції: `print`, `input`, `len`, `typeOf`, `error`, `errorMessage`, `fromString`, `toInt`
- методи вбудованих типів: для `list`, `string`, `bytes`, `dict`

## Внутрішні вбудовані функції stdlib

Стандартна бібліотека Avenir делегує частину операцій внутрішнім вбудованим функціям:

- `__builtin_fs_*`
- `__builtin_socket_*`
- `__builtin_json_*`
- `__builtin_http_*`

## Обробка помилок

VM використовує єдину модель помилок: після компіляції помилки представлені
як значення `error` і можуть бути перехоплені через `try`/`catch`.

Необроблені винятки підіймаються стеком викликів до верхнього рівня,
після чого виконання завершується повідомленням про помилку.

## Керування пам'яттю

Пам'ять для стекових значень, замикань, структур і списків керується автоматично.

## Продуктивність

VM орієнтована на:

- швидку диспетчеризацію інструкцій
- компактний байткод
- ефективні виклики функцій
