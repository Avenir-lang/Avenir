# Огляд мови Avenir

Avenir — статично типізована мова програмування, призначена для створення надійного ПЗ. Мова поєднує чистий, читабельний синтаксис із строгою типобезпекою та повноцінним середовищем виконання.

## Ключові можливості

### Система типів

- **Статична типізація**: усі типи перевіряються під час компіляції
- **Виведення типів**: локальне виведення типів для `var name = expr;` та виведення при узагальнених викликах
- **Структурні та номінальні типи**: списки використовують структурну типізацію, структури — номінальну
- **Union-типи**: підтримка об'єднань типів (`<int|string>`)
- **Опціональні типи**: опціональні значення (`T?`) для nullable-типів
- **Типи функцій**: first-class типи функцій

### Основні типи

- **Примітиви**: `int`, `float`, `string`, `bool`, `bytes`
- **Колекції**: `list<T>` та `dict<K, V>` (або `dict<V>`)
- **Структури**: користувацькі типи з іменованими полями
- **Функції**: first-class функції та замикання
- **Error**: вбудований тип помилок для обробки виключень

### Система структур

- **Контроль видимості**: ключове слово `pub` для публічних структур і полів
- **Контроль мутабельності**: `mut struct` для мутабельних структур, перевизначення `mut` на рівні полів
- **Значення за замовчуванням**: поля структур можуть мати значення за замовчуванням часу компіляції
- **Методи**: як методи екземпляра, так і статичні
- **Присвоєння полів**: мутація на місці для мутабельних полів

### Функції

- **Іменовані функції**: оголошення функцій верхнього рівня
- **Літерали функцій**: анонімні функції та замикання
- **Параметри за замовчуванням**: параметри функцій можуть мати значення за замовчуванням
- **Іменовані аргументи**: виклики функцій підтримують іменовані аргументи
- **Методи**: методи екземпляра з receiver-ами, статичні методи на типах
- **Декоратори**: декоратори функцій із синтаксисом `@` для метапрограмування
- **Дженеріки**: узагальнені функції з явними аргументами типів

### Структура програми

- **Вирази верхнього рівня**: вирази-оператори на рівні модуля для ініціалізації
- **Система модулів**: модульна архітектура на основі імпортів
- **Точка входу**: функція `main()` як точка входу програми

### Керування потоком

- **Умови**: оператори `if`/`else` з опціональним ланцюжком
- **Цикли**: `while`, C-подібний `for` та `for...in`
- **Switch**: `switch` на основі значень з `case` і `default` (без fallthrough)
- **Виключення**: `try`/`catch` з типізованими catch-клаузами для обробки помилок
- **Керування циклами**: `break` і `continue`
- **Відкладені виклики**: `defer` виконує вирази виклику при поверненні з функції (LIFO)

### Система модулів

- **Оголошення пакетів**: кожен файл починається з оголошення `pckg`
- **Імпорти**: імпорти на основі модулів з опціональними аліасами
- **Зіставлення файл-структура**: файли зі структурами мають відповідати іменам структур
- **Імпорт на основі папок**: підтримка синтаксису `A/A.av` → `import module.A`
- **Видимість**: контроль видимості на рівні модуля через `pub`
- **Резолв модулів**: подвійний резолв (на основі папок і плоских файлів)

### Середовище виконання

- **Віртуальна машина**: стекова VM з байткодом
- **Вбудовані функції**: базові функції для I/O, колекцій, рядків, байтів
- **Вбудовані методи**: методи на вбудованих типах (рядки, списки, байти)
- **Замикання**: повна підтримка замикань та лексичної області видимості

## Філософія

Avenir розроблена для:

- **Типобезпеки**: виявлення помилок під час компіляції, а не виконання
- **Явності**: чіткий, читабельний код з явними типами
- **Практичності**: можливості для реальної розробки ПЗ
- **Продуктивності**: компіляція в байткод з ефективною VM

## Приклад

```avenir
pckg main;

struct Point {
    x | int
    y | int
}

fun (p | Point).sum() | int {
    return p.x + p.y;
}

fun main() | void {
    var p | Point = Point{x = 3, y = 4};
    var s | int = p.sum();
    print(s);
}
```
