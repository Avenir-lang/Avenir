# Часті запитання

## Загальні питання

### Що таке Avenir?

Avenir — статично типізована мова програмування, орієнтована на створення
надійного ПЗ. Вона пропонує чистий синтаксис, строгу типізацію та середовище
виконання на базі віртуальної машини.

### Які платформи підтримує Avenir?

Avenir реалізовано на Go і він працює на платформах, які підтримує Go
(Linux, macOS, Windows та інші).

### Чи готовий Avenir до продакшену?

Мова перебуває в активній розробці. Деякі можливості ще можуть бути не повністю
реалізовані або змінитися.

## Можливості мови

### Чи підтримує Avenir дженеріки?

Так. Avenir підтримує явні дженеріки для користувацьких структур і функцій.

```avenir
struct Box<T> {
    value | T
}

fun identity<T>(x | T) | T {
    return x;
}

var b | Box<int> = Box<int>{value = 1};
var v | int = identity<int>(10);
```

Аргументи типів зараз указуються явно в місцях виклику та використання типів.

### Чи підтримує Avenir успадкування?

Ні. В Avenir використовуються композиція та методи, а не успадкування.

### Чи підтримує Avenir інтерфейси?

Так. Avenir підтримує структурну типізацію через інтерфейси. Тип реалізує
інтерфейс неявно, якщо має всі необхідні методи з відповідними сигнатурами.
Детальніше див. [Типи](types.md).

### Чи підтримує Avenir перевантаження операторів?

Ні. Оператори мають фіксовану семантику.

### Чи підтримує Avenir перевантаження функцій?

Ні. Ім'я функції має бути унікальним у межах області видимості.

### Чи підтримує Avenir опціональний ланцюжок?

Поки що ні. Опціональні типи вже існують, але опціональний ланцюжок ще не реалізовано.

### Чи підтримує Avenir зіставлення зі зразком?

Поки що ні. Зіставлення зі зразком ще не реалізовано.

## Система типів

### Чи підтримується виведення типів?

Ні. Avenir вимагає явних анотацій типів. Виведення типів не підтримується.

### Чи можна всюди використовувати `any`?

Технічно можна, але зазвичай краще використовувати конкретні типи заради
безпеки типів.

### Структури — це типи-значення чи посилальні типи?

Структури є типами-значеннями. Присвоєння створює копію. При цьому змінювані поля можуть
змінюватися на місці.

### Як працювати з порожніми значеннями?

Використовуйте опціональні типи (`T?`) разом із `some()` і `none()`:

```avenir
var x | int? = some(42);
var y | int? = none;
```

## Модулі та імпорти

### Як створити модуль?

Створіть файл з оголошенням пакета:

```avenir
pckg mymodule;
```

### Як імпортувати модуль?

Використовуйте `import`:

```avenir
import mymodule;
```

### Що таке правило відповідності файла і структури?

Якщо файл містить структури, **хоча б одна структура повинна мати те саме ім'я,
що й файл** без розширення `.av`. Наприклад:

- `Point.av` повинен містити `struct Point`, якщо в ньому є структури
- файли без структур теж можна імпортувати

### Як працюють папкові імпорти?

Якщо папка `A` містить файл `A.av`, її можна імпортувати за назвою папки:

```
Тека: geometry/
Файл: geometry/geometry.av
Імпорт: import geometry;
```

Компілятор автоматично резолвить `import geometry` у `geometry/geometry.av`.

### Чи можна робити циклічні імпорти?

Ні. Цикли імпортів виявляються і вважаються помилкою.

### Де розташовані модулі?

Модулі шукаються за файловими шляхами:

- **Стандартні модулі** (`std.*`): шукаються в каталозі `std/`
- **Модулі застосунку**: шукаються відносно кореня проєкту

Спочатку компілятор пробує папковий імпорт (`module/A/A.av`), потім — плоский файл (`module/A.av`).

## Функції та методи

### У чому різниця між функцією та методом?

Функція — це самостійна функція. Метод — це функція, пов'язана з типом і
оголошена як метод екземпляра або статичний метод.

### Чи можна використовувати параметри за замовчуванням?

Так. Параметри функції можуть мати значення за замовчуванням:

```avenir
fun greet(name | string, greeting | string = "Hello") | void {
    // ...
}
```

### Чи можна використовувати іменовані аргументи?

Так. Виклики функцій підтримують іменовані аргументи:

```avenir
create_point(x = 10, y = 20);
```

### Чи можна повертати кілька значень?

Не напряму. Для цього використовуйте структуру або список:

```avenir
struct Result {
    value | int
    error | error?
}

fun compute() | Result {
    return Result{value = 42, error = none};
}
```

## Керування потоком

### Чи можна використовувати `break` усередині `if`?

Ні. `break` можна використовувати лише всередині циклів.

### Чи можна використовувати `continue`?

Поки що ні. `continue` ще не реалізовано.

### Чи можна використовувати `switch`?

Поки що ні. Натомість використовуйте ланцюжки `if-else`.

### Чи можна використовувати `defer`?

Поки що ні. `defer` ще не реалізовано.

## Обробка помилок

### Як обробляти помилки?

Використовуйте `try-catch`:

```avenir
try {
} catch (e | error) {
}
```

### Чи можна повертати помилки замість `throw`?

Поки що ні. Помилки мають викидатися як винятки.

### Чи можна створювати власні типи помилок?

Поки що ні. Усі помилки повинні мати тип `error`.

## Продуктивність

### Чи швидкий Avenir?

VM оптимізована для виконання, але скомпільований нативний код був би швидшим.

### Чим Avenir відрізняється від інших мов?

Avenir орієнтований на надійність і ясність. Продуктивність враховується, але не
є головною метою.

## Розробка

### Як налагоджувати програми на Avenir?

Використовуйте `print` для налагодження. Повноцінного налагоджувача поки немає.

### Як тестувати програми на Avenir?

Пишіть тестові програми та запускайте їх. Окремого фреймворку тестування поки немає.

### Як профілювати програми на Avenir?

Поки спеціалізовані засоби профілювання недоступні.
