# Структури

Структури — це користувацькі типи з іменованими полями. Для структур
використовується номінальна типізація: дві структури рівні лише тоді, коли
вони мають однакове ім'я.

## Оголошення структур

Структури оголошуються за допомогою ключового слова `struct`:

```avenir
struct Point {
    x | int
    y | int
}
```

### Узагальнені структури

Структури можуть оголошувати параметри типів після імені структури:

```avenir
struct Box<T> {
    value | T
}

struct Pair<T, U> {
    first | T
    second | U
}
```

Аргументи типів у місцях використання вказуються явно:

```avenir
var n | Box<int> = Box<int>{value = 42};
var p | Pair<int, string> = Pair<int, string>{first = 1, second = "one"};
```

## Публічні структури

Структури можна зробити публічними за допомогою ключового слова `pub`:

```avenir
pub struct Point {
    x | int
    y | int
}
```

Публічні структури доступні з інших модулів. Приватні структури (`без pub`)
доступні лише всередині того самого модуля.

## Змінювані структури

Структури можна позначити як змінювані за допомогою ключового слова `mut`:

```avenir
mut struct Point {
    x | int
    y | int
}
```

Змінювані структури допускають присвоєння полям. Незмінювані структури (`без mut`)
не допускають присвоєння полям.

## Поля

### Оголошення полів

Поля оголошуються з іменем і типом:

```avenir
struct Point {
    x | int
    y | int
}
```

### Публічні поля

Поля можна зробити публічними за допомогою ключового слова `pub`:

```avenir
pub struct Point {
    pub x | int
    pub y | int
}
```

Публічні поля доступні з інших модулів. Приватні поля (`без pub`) доступні
лише всередині того самого модуля або з методів тієї самої структури.

**Примітка**: Публічні поля можна оголошувати лише в публічних структурах.
Приватні структури не можуть мати публічних полів.

### Змінювані поля

Поля можна позначити як змінювані за допомогою ключового слова `mut`:

```avenir
struct Point {
    mut x | int
    y | int
}
```

Змінювані поля допускають присвоєння навіть у незмінюваних структурах. `mut`
на рівні поля перевизначає змінюваність на рівні структури.

### Правила змінюваності полів

Присвоєння полю дозволене лише тоді, коли поле змінюване. Змінюваність поля
визначається так:

1. Якщо структура оголошена як `mut struct`, усі поля за замовчуванням змінювані
2. Якщо поле позначене як `mut`, воно змінюване незалежно від змінюваності структури
3. Інакше поле є незмінюваним

Приклади:

```avenir
// Незмінювана структура, незмінюване поле
struct Point {
    x | int
}
// Point{x = 10}.x = 20;

// Незмінювана структура, змінюване поле
struct Point {
    mut x | int
}
// Point{x = 10}.x = 20;

// Змінювана структура, незмінюване поле
mut struct Point {
    x | int
}
// Point{x = 10}.x = 20;

// Змінювана структура, змінюване поле
mut struct Point {
    mut x | int
}
// Point{x = 10}.x = 20;
```

### Значення за замовчуванням

Поля можуть мати значення за замовчуванням:

```avenir
struct Config {
    host | string = "localhost"
    port | int = 8080
}
```

Значення за замовчуванням повинні бути константами часу компіляції. Поля зі
значеннями за замовчуванням можна опускати в літералах структур.

## Літерали структур

Літерали структур створюють значення структур:

```avenir
var p | Point = Point{x = 10, y = 20};
```

Літерали узагальнених структур містять явні аргументи типів:

```avenir
var b | Box<int> = Box<int>{value = 99};
```

Поля ініціалізуються за допомогою іменованої ініціалізації: `field = value`.

### Часткова ініціалізація

Поля зі значеннями за замовчуванням можна опускати:

```avenir
struct Config {
    host | string = "localhost"
    port | int = 8080
}

var c | Config = Config{port = 9000};
```

Поля без значень за замовчуванням потрібно вказувати обов'язково.

### Присвоєння полям

Для змінюваних структур або змінюваних полів присвоєння можливе після
ініціалізації:

```avenir
mut struct Point {
    x | int
    y | int
}

var p | Point = Point{x = 10, y = 20};
p.x = 30;
p.y = 40;
```

Присвоєння виконується на місці, без копіювання.

## Доступ до полів

Доступ до полів здійснюється через крапкову нотацію:

```avenir
var p | Point = Point{x = 10, y = 20};
var x | int = p.x;
var y | int = p.y;
```

## Система типів

Для структур використовується номінальна типізація: дві структури рівні лише
тоді, коли мають однакове ім'я, незалежно від структури полів.

```avenir
struct Point {
    x | int
    y | int
}

struct Coordinate {
    x | int
    y | int
}

// Point і Coordinate — різні типи, навіть якщо вони мають однакові поля
```
