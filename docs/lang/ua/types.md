# Типи

Avenir — статично типізована мова. Усі типи перевіряються на етапі компіляції.

## Примітивні типи

### `int`

64-бітне знакове ціле.

```avenir
var x | int = 42;
```

### `float`

64-бітне число з плаваючою точкою.

```avenir
var x | float = 3.14;
```

### `string`

Рядок символів.

```avenir
var s | string = "Hello";
```

### `bool`

Логічне значення (`true` або `false`).

```avenir
var b | bool = true;
```

### `bytes`

Послідовність байтів.

```avenir
var data | bytes = b"bytes";
```

### `void`

Позначає відсутність значення, що повертається.

```avenir
fun print_hello() | void {
    print("Hello");
}
```

### `any`

Позначає довільний тип.

```avenir
var x | any = 42;
```

### `error`

Тип помилки для обробки винятків.

```avenir
var e | error = error("something went wrong");
```

## Інтерфейси

Інтерфейси задають набір сигнатур методів. В Avenir використовується **структурна типізація**: тип автоматично задовольняє інтерфейс, якщо має всі потрібні методи з сумісними сигнатурами.

### Оголошення інтерфейсу

Інтерфейси оголошуються за допомогою ключового слова `interface`:

```avenir
interface Stringer {
    fun toString() | string
}
```

Інтерфейси можуть бути публічними:

```avenir
pub interface Writer {
    fun write(data | string) | void
    fun flush() | void
}
```

### Структурна типізація

Тип задовольняє інтерфейс **неявно**, якщо має всі необхідні методи. Ключового слова `implements` немає:

```avenir
interface Stringer {
    fun toString() | string
}

struct Point {
    x | int
    y | int
}

fun (p | Point).toString() | string {
    return "Point";
}

fun main() | void {
    var s | Stringer = Point{x = 1, y = 2};
    var str | string = s.toString();
}
```

### Методи інтерфейсу

Методи інтерфейсу оголошуються повною сигнатурою:

```avenir
interface Writer {
    fun write(data | string) | void
    fun flush() | void
}
```

Сигнатури повинні точно збігатися:
- ім'я методу має збігатися
- кількість і типи параметрів мають збігатися
- тип повернення має збігатися

### Використання інтерфейсів як типів

Інтерфейси можна використовувати як типи:

```avenir
fun printValue(x | Stringer) | void {
    print(x.toString());
}
```

### Сумісність методів

Метод задовольняє метод інтерфейсу, якщо:
- ім'я збігається
- кількість і порядок параметрів збігаються
- типи параметрів збігаються
- тип повернення збігається

Лише **методи екземпляра** можуть задовольняти інтерфейси. Статичні методи не підходять.

### Відповідність вбудованих типів

Вбудовані типи також можуть задовольняти інтерфейси, якщо мають потрібні методи:

```avenir
interface Length {
    fun length() | int
}

fun main() | void {
    var l | Length = "hello";  // string has length() method
    var len | int = l.length();
}
```

### Правила видимості

- публічні інтерфейси можуть вимагати лише публічні методи
- приватні методи не можуть задовольняти публічні інтерфейси між модулями
- у межах одного модуля приватні методи можуть задовольняти публічні інтерфейси

### Повідомлення про помилки

Якщо тип не задовольняє інтерфейс, компілятор повертає детальне повідомлення:

```
type Point does not satisfy interface Stringer: missing methods: toString() | string
```

або

```
type Point does not satisfy interface Stringer: incompatible method signatures: toString: expected () | string, got () | int
```

## Складені типи

### Користувацькі узагальнені типи

Користувацькі структури можуть бути узагальненими:

```avenir
struct Box<T> {
    value | T
}

var a | Box<int> = Box<int>{value = 1};
var b | Box<string> = Box<string>{value = "hi"};
```

Аргументи типів указуються явно.

### Списки

Списки — це послідовності значень. Вони можуть бути однорідними або змішаними за типами елементів.

```avenir
var numbers | list<int> = [1, 2, 3];
var mixed | list<int, string> = [1, "two", 3];
```

Списки використовують структурну типізацію.

`list<...>` — вбудований параметричний тип, а не користувацька узагальнена структура.

### Словники

Словники відображають рядкові ключі у значення типу `T` і записуються як `dict<T>`.

```avenir
var user | dict<any> = { name: "Alex", "age": 30 };
var scores | dict<int> = { alice: 10, bob: 12 };
```

Ключі завжди є рядками; значення мають бути сумісні з `T`.
Якщо ключ може бути відсутнім, використовуйте `dict.get()`, він повертає `T?`.

`dict<T>` — також вбудований параметричний тип.

### Структури

Користувацькі типи з іменованими полями.

```avenir
struct Point {
    x | int
    y | int
}
```

Структури використовують номінальну типізацію.

### Функції

Типи функцій задають типи параметрів і результат:

```avenir
fun (int) | int
fun (int, string) | bool
```

### Опціональні типи

Опціональні типи дозволяють представляти відсутність значення:

```avenir
var x | int? = some(42);
var y | int? = none;
```

Опціональні типи записуються як `T?`. `none` — єдине null-подібне значення в мові.

### Типи `Future`

Результати async-функцій моделюються як `Future<T>`, а `await` витягає з них `T`.

### Об'єднані типи

Об'єднані типи дозволяють значенню бути одним із кількох типів.

```avenir
var value | <int|string> = 42;
value = "hello";
```

Об'єднані типи записуються в кутових дужках: `<T1|T2|...>`.

## Система типів

### Структурна і номінальна типізація

- **Списки**: структурна типізація
- **Структури**: номінальна типізація
- **Функції**: структурна типізація

### Перевірка типів

Усі типи перевіряються на етапі компіляції.

### Сумісність присвоєння

Значення типу `T` можна присвоїти змінній типу `U`, якщо:

1. `T` і `U` збігаються
2. `T` сумісний з `U`
3. `T` є підтипом `U`, якщо це застосовно

## Анотації типів

Типи вказуються через роздільник `|`:

```avenir
var name | type = value;
```

Для параметрів і значення, що повертається функцією:

```avenir
fun name(param1 | type1, param2 | type2) | returnType {
    // ...
}
```

## Виведення типів

Avenir вимагає явних анотацій типів. Виведення типів і виведення аргументів узагальнених типів поки не підтримується.
