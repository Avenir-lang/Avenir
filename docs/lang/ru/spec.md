# Спецификация языка (краткая)

Этот документ даёт краткое нормативное описание базового синтаксиса и семантики
Avenir. За подробными объяснениями и примерами смотрите другие файлы в этом
каталоге.

## Лексическая структура

- идентификаторы: буквы/`_`, затем буквы/цифры/`_`
- ключевые слова: `pckg`, `import`, `fun`, `struct`, `pub`, `mut`, `var`, `if`, `else`, `while`, `for`, `in`, `return`, `break`, `try`, `catch`, `throw`, `true`, `false`, `none`, `some`, `interface`
- литералы: `int`, `float`, `string`, `bytes`, `bool`, `none`, `some(...)`

## Типы

- примитивы: `int`, `float`, `string`, `bool`, `bytes`, `void`, `any`, `error`
- составные типы: `list<T>`, `dict<T>`, типы функций `fun(...) | T`
- опциональные типы: `T?`
- объединённые типы: `<T1|T2|...>`
- типы структур и интерфейсов
- объявления обобщённых типов у структур и функций: `Name<T, U, ...>`
- использования обобщённых типов требуют явных аргументов типов

## Выражения

- арифметические, сравнительные и логические операторы
- конкатенация строк через `+` разрешена только для `string + string`
- индексирование: `list[int]`, `bytes[int]`, `dict[string]`
- доступ к полям и методам: `expr.field`, `expr.method(...)`
- вызовы обобщённых функций: `fn<T, U>(...)`

## Операторы

- объявления переменных: `var name | Type = expr;`
- присваивание: `name = expr;`
- `if`, `while`, `for`, `for (item in list)`
- `return`, `break`, `throw`, `try/catch`

## Модули

- каждый файл объявляет пакет через `pckg`
- импорты используют точечные пути: `import std.net as net;`
- папочные модули разрешаются в `path/path.av` с правилом соответствия файл/структура

## Семантика среды выполнения

- все ошибки времени выполнения выбрасываются как значения `error` и могут быть перехвачены с помощью `try / catch`
- встроенные функции и методы являются частью ядра языка и вызываются напрямую виртуальной машиной

Подробности смотрите в `syntax.md`, `types.md`, `operators.md`, `builtins.md` и `docs/std/`.
