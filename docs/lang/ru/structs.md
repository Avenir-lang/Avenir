# Структуры

Структуры — это пользовательские типы с именованными полями. Для структур
используется номинальная типизация: две структуры равны только в том случае,
если у них одинаковое имя.

## Объявление структур

Структуры объявляются с помощью ключевого слова `struct`:

```avenir
struct Point {
    x | int
    y | int
}
```

### Обобщённые структуры

Структуры могут объявлять параметры типов после имени структуры:

```avenir
struct Box<T> {
    value | T
}

struct Pair<T, U> {
    first | T
    second | U
}
```

Аргументы типов в местах использования указываются явно:

```avenir
var n | Box<int> = Box<int>{value = 42};
var p | Pair<int, string> = Pair<int, string>{first = 1, second = "one"};
```

## Публичные структуры

Структуры можно сделать публичными с помощью ключевого слова `pub`:

```avenir
pub struct Point {
    x | int
    y | int
}
```

Публичные структуры доступны из других модулей. Приватные структуры
(`без pub`) доступны только внутри того же модуля.

## Изменяемые структуры

Структуры можно пометить как изменяемые с помощью ключевого слова `mut`:

```avenir
mut struct Point {
    x | int
    y | int
}
```

Изменяемые структуры допускают присваивание полям. Неизменяемые структуры
(`без mut`) не допускают присваивание полям.

## Поля

### Объявление полей

Поля объявляются с именем и типом:

```avenir
struct Point {
    x | int
    y | int
}
```

### Публичные поля

Поля можно сделать публичными с помощью ключевого слова `pub`:

```avenir
pub struct Point {
    pub x | int
    pub y | int
}
```

Публичные поля доступны из других модулей. Приватные поля (`без pub`) доступны
только внутри того же модуля или из методов той же структуры.

**Примечание**: Публичные поля можно объявлять только в публичных структурах.
Приватные структуры не могут иметь публичных полей.

### Изменяемые поля

Поля можно пометить как изменяемые с помощью ключевого слова `mut`:

```avenir
struct Point {
    mut x | int
    y | int
}
```

Изменяемые поля допускают присваивание даже в неизменяемых структурах. `mut`
на уровне поля переопределяет изменяемость на уровне структуры.

### Правила изменяемости полей

Присваивание полю разрешено только в том случае, если поле изменяемо.
Изменяемость поля определяется так:

1. Если структура объявлена как `mut struct`, все поля по умолчанию изменяемые
2. Если поле помечено как `mut`, оно изменяемо независимо от изменяемости структуры
3. В противном случае поле неизменяемо

Примеры:

```avenir
// Неизменяемая структура, неизменяемое поле
struct Point {
    x | int
}
// Point{x = 10}.x = 20;

// Неизменяемая структура, изменяемое поле
struct Point {
    mut x | int
}
// Point{x = 10}.x = 20;

// Изменяемая структура, неизменяемое поле
mut struct Point {
    x | int
}
// Point{x = 10}.x = 20;

// Изменяемая структура, изменяемое поле
mut struct Point {
    mut x | int
}
// Point{x = 10}.x = 20;
```

### Значения по умолчанию

Поля могут иметь значения по умолчанию:

```avenir
struct Config {
    host | string = "localhost"
    port | int = 8080
}
```

Значения по умолчанию должны быть константами времени компиляции. Поля со
значениями по умолчанию можно опускать в литералах структур.

## Литералы структур

Литералы структур создают значения структур:

```avenir
var p | Point = Point{x = 10, y = 20};
```

Литералы обобщённых структур содержат явные аргументы типов:

```avenir
var b | Box<int> = Box<int>{value = 99};
```

Поля инициализируются через именованную инициализацию: `field = value`.

### Частичная инициализация

Поля со значениями по умолчанию можно опускать:

```avenir
struct Config {
    host | string = "localhost"
    port | int = 8080
}

var c | Config = Config{port = 9000};
```

Поля без значений по умолчанию должны быть указаны обязательно.

### Присваивание полям

Для изменяемых структур или изменяемых полей присваивание возможно после
инициализации:

```avenir
mut struct Point {
    x | int
    y | int
}

var p | Point = Point{x = 10, y = 20};
p.x = 30;
p.y = 40;
```

Присваивание выполняется на месте, без копирования.

## Доступ к полям

Доступ к полям осуществляется через точечную нотацию:

```avenir
var p | Point = Point{x = 10, y = 20};
var x | int = p.x;
var y | int = p.y;
```

## Система типов

Для структур используется номинальная типизация: две структуры равны только в
том случае, если у них одинаковое имя, независимо от структуры полей.

```avenir
struct Point {
    x | int
    y | int
}

struct Coordinate {
    x | int
    y | int
}

// Point и Coordinate — разные типы, даже если у них одинаковые поля
```
