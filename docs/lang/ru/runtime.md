# Среда выполнения

Программы Avenir выполняются на виртуальной машине (VM). VM стековая и
исполняет байткод.

## Модель выполнения

Программа проходит этапы:

1. парсинг исходного кода
2. проверка типов
3. компиляция в байткод (IR)
4. исполнение в VM

## Виртуальная машина

VM использует стек операндов и стек вызовов.

### Стек

Стек хранит:

- операнды выражений
- аргументы и возвращаемые значения функций
- локальные переменные

### Вызовы функций

При вызове VM:

1. кладёт аргументы в стек
2. создаёт новый фрейм
3. исполняет тело функции
4. снимает результат
5. восстанавливает предыдущий фрейм

### Асинхронность: `async` и `await`

Avenir поддерживает асинхронное выполнение через `async` и `await`.

- вызов async-функции возвращает значение `Future`
- `await` получает результат `Future`
- среда выполнения использует однопоточный планировщик и цикл событий
- если значение `Future` ещё не готово, текущая задача приостанавливается и возобновляется позже

### Замыкания

Замыкания захватывают переменные внешней области видимости. Захваченные
значения хранятся в массиве захваченных значений.

## Встроенные функции и методы

Встроенные функции реализованы на Go и вызываются напрямую из VM.

- функции: `print`, `input`, `len`, `typeOf`, `error`, `errorMessage`, `fromString`, `toInt`
- методы встроенных типов: для `list`, `string`, `bytes`, `dict`

## Внутренние встроенные функции stdlib

Стандартная библиотека Avenir делегирует часть операций внутренним встроенным функциям:

- `__builtin_fs_*`
- `__builtin_socket_*`
- `__builtin_json_*`
- `__builtin_http_*`

## Обработка ошибок

VM использует единую модель ошибок: после компиляции ошибки представлены
как значения `error` и могут быть перехвачены через `try`/`catch`.

Необработанные исключения поднимаются по стеку вызовов до верхнего уровня,
после чего выполнение завершается сообщением об ошибке.

## Управление памятью

Память для стековых значений, замыканий, структур и списков управляется
автоматически.

## Производительность

VM ориентирована на:

- быструю диспетчеризацию инструкций
- компактный байткод
- эффективные вызовы функций
