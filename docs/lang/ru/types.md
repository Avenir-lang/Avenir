# Типы

Avenir — статически типизированный язык. Все типы проверяются на этапе компиляции.

## Примитивные типы

### `int`

64-битное знаковое целое.

```avenir
var x | int = 42;
```

### `float`

64-битное число с плавающей точкой.

```avenir
var x | float = 3.14;
```

### `string`

Строка символов.

```avenir
var s | string = "Hello";
```

### `bool`

Логическое значение (`true` или `false`).

```avenir
var b | bool = true;
```

### `bytes`

Последовательность байтов.

```avenir
var data | bytes = b"bytes";
```

### `void`

Обозначает отсутствие возвращаемого значения.

```avenir
fun print_hello() | void {
    print("Hello");
}
```

### `any`

Обозначает произвольный тип.

```avenir
var x | any = 42;
```

### `error`

Тип ошибки для обработки исключений.

```avenir
var e | error = error("something went wrong");
```

## Интерфейсы

Интерфейсы задают набор сигнатур методов. В Avenir используется **структурная типизация**: тип автоматически удовлетворяет интерфейсу, если у него есть все нужные методы с совместимыми сигнатурами.

### Объявление интерфейса

Интерфейсы объявляются с помощью ключевого слова `interface`:

```avenir
interface Stringer {
    fun toString() | string
}
```

Интерфейсы могут быть публичными:

```avenir
pub interface Writer {
    fun write(data | string) | void
    fun flush() | void
}
```

### Структурная типизация

Тип удовлетворяет интерфейсу **неявно**, если у него есть все необходимые методы. Ключевого слова `implements` нет:

```avenir
interface Stringer {
    fun toString() | string
}

struct Point {
    x | int
    y | int
}

fun (p | Point).toString() | string {
    return "Point";
}

fun main() | void {
    var s | Stringer = Point{x = 1, y = 2};
    var str | string = s.toString();
}
```

### Методы интерфейса

Методы интерфейса объявляются полной сигнатурой:

```avenir
interface Writer {
    fun write(data | string) | void
    fun flush() | void
}
```

Сигнатуры должны совпадать точно:
- имя метода должно совпадать
- количество и типы параметров должны совпадать
- возвращаемый тип должен совпадать

### Использование интерфейсов как типов

Интерфейсы можно использовать как типы:

```avenir
fun printValue(x | Stringer) | void {
    print(x.toString());
}
```

### Совместимость методов

Метод удовлетворяет методу интерфейса, если:
- имя совпадает
- количество и порядок параметров совпадают
- типы параметров совпадают
- возвращаемый тип совпадает

Только **методы экземпляра** могут удовлетворять интерфейсам. Статические методы не подходят.

### Соответствие встроенных типов

Встроенные типы тоже могут удовлетворять интерфейсам, если имеют нужные методы:

```avenir
interface Length {
    fun length() | int
}

fun main() | void {
    var l | Length = "hello";  // string has length() method
    var len | int = l.length();
}
```

### Правила видимости

- публичные интерфейсы могут требовать только публичные методы
- приватные методы не могут удовлетворять публичным интерфейсам между модулями
- внутри одного модуля приватные методы могут удовлетворять публичным интерфейсам

### Сообщения об ошибках

Если тип не удовлетворяет интерфейсу, компилятор возвращает подробное сообщение:

```
type Point does not satisfy interface Stringer: missing methods: toString() | string
```

или

```
type Point does not satisfy interface Stringer: incompatible method signatures: toString: expected () | string, got () | int
```

## Составные типы

### Пользовательские обобщённые типы

Пользовательские структуры могут быть обобщёнными:

```avenir
struct Box<T> {
    value | T
}

var a | Box<int> = Box<int>{value = 1};
var b | Box<string> = Box<string>{value = "hi"};
```

Аргументы типов указываются явно.

### Списки

Списки — это последовательности значений. Они могут быть однородными или смешанными по типам элементов.

```avenir
var numbers | list<int> = [1, 2, 3];
var mixed | list<int, string> = [1, "two", 3];
```

Списки используют структурную типизацию.

`list<...>` — встроенный параметрический тип, а не пользовательская обобщённая структура.

### Словари

Словари отображают строковые ключи в значения типа `T` и записываются как `dict<T>`.

```avenir
var user | dict<any> = { name: "Alex", "age": 30 };
var scores | dict<int> = { alice: 10, bob: 12 };
```

Ключи всегда строки; значения должны быть совместимы с `T`.
Если ключ может отсутствовать, используйте `dict.get()`, он возвращает `T?`.

`dict<T>` — тоже встроенный параметрический тип.

### Структуры

Пользовательские типы с именованными полями.

```avenir
struct Point {
    x | int
    y | int
}
```

Структуры используют номинальную типизацию.

### Функции

Типы функций задают типы параметров и результат:

```avenir
fun (int) | int
fun (int, string) | bool
```

### Опциональные типы

Опциональные типы позволяют представлять отсутствие значения:

```avenir
var x | int? = some(42);
var y | int? = none;
```

Опциональные типы записываются как `T?`. `none` — единственное null-подобное значение в языке.

### Типы `Future`

Результаты async-функций моделируются как `Future<T>`, а `await` извлекает из них `T`.

### Объединённые типы

Объединённые типы позволяют значению быть одним из нескольких типов.

```avenir
var value | <int|string> = 42;
value = "hello";
```

Объединённые типы записываются в угловых скобках: `<T1|T2|...>`.

## Система типов

### Структурная и номинальная типизация

- **Списки**: структурная типизация
- **Структуры**: номинальная типизация
- **Функции**: структурная типизация

### Проверка типов

Все типы проверяются на этапе компиляции.

### Совместимость присваивания

Значение типа `T` можно присвоить переменной типа `U`, если:

1. `T` и `U` совпадают
2. `T` совместим с `U`
3. `T` является подтипом `U`, если это применимо

## Аннотации типов

Типы указываются через разделитель `|`:

```avenir
var name | type = value;
```

Для параметров и возвращаемого значения функции:

```avenir
fun name(param1 | type1, param2 | type2) | returnType {
    // ...
}
```

## Вывод типов

Avenir требует явных аннотаций типов. Вывод типов и вывод аргументов обобщённых типов пока не поддерживаются.
