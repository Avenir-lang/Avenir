# Часто задаваемые вопросы

## Общие вопросы

### Что такое Avenir?

Avenir — статически типизированный язык программирования, ориентированный на
создание надёжного ПО. Он предлагает чистый синтаксис, строгую типизацию и
среду выполнения на базе виртуальной машины.

### Какие платформы поддерживает Avenir?

Avenir реализован на Go и работает на платформах, которые поддерживает Go
(Linux, macOS, Windows и другие).

### Готов ли Avenir к продакшену?

Язык находится в активной разработке. Некоторые возможности ещё могут быть
неполностью реализованы или измениться.

## Возможности языка

### Поддерживает ли Avenir дженерики?

Да. Avenir поддерживает явные дженерики для пользовательских структур и функций.

```avenir
struct Box<T> {
    value | T
}

fun identity<T>(x | T) | T {
    return x;
}

var b | Box<int> = Box<int>{value = 1};
var v | int = identity<int>(10);
```

Аргументы типов сейчас указываются явно в местах вызова и использования типов.

### Поддерживает ли Avenir наследование?

Нет. В Avenir используются композиция и методы, а не наследование.

### Поддерживает ли Avenir интерфейсы?

Да. Avenir поддерживает структурную типизацию через интерфейсы. Тип реализует
интерфейс неявно, если у него есть все требуемые методы с подходящими
сигнатурами. Подробнее см. [Типы](types.md).

### Поддерживает ли Avenir перегрузку операторов?

Нет. У операторов фиксированная семантика.

### Поддерживает ли Avenir перегрузку функций?

Нет. Имя функции должно быть уникальным в пределах области видимости.

### Поддерживает ли Avenir опциональную цепочку?

Пока нет. Опциональные типы уже существуют, но опциональная цепочка ещё не реализована.

### Поддерживает ли Avenir сопоставление с образцом?

Пока нет. Сопоставление с образцом ещё не реализовано.

## Система типов

### Поддерживается ли вывод типов?

Нет. Avenir требует явных аннотаций типов. Вывод типов не поддерживается.

### Можно ли везде использовать `any`?

Технически можно, но обычно лучше использовать конкретные типы ради безопасности
типов.

### Структуры — это типы-значения или ссылочные типы?

Структуры являются типами-значениями. Присваивание создаёт копию. При этом изменяемые поля
могут изменяться на месте.

### Как работать с пустыми значениями?

Используйте опциональные типы (`T?`) вместе с `some()` и `none()`:

```avenir
var x | int? = some(42);
var y | int? = none;
```

## Модули и импорты

### Как создать модуль?

Создайте файл с объявлением пакета:

```avenir
pckg mymodule;
```

### Как импортировать модуль?

Используйте `import`:

```avenir
import mymodule;
```

### Что такое правило соответствия файла и структуры?

Если файл содержит структуры, **хотя бы одна структура должна иметь то же имя,
что и файл** без расширения `.av`. Например:

- `Point.av` должен содержать `struct Point`, если в нём есть структуры
- файлы без структур тоже можно импортировать

### Как работают папочные импорты?

Если папка `A` содержит файл `A.av`, её можно импортировать по имени папки:

```
Папка: geometry/
Файл: geometry/geometry.av
Импорт: import geometry;
```

Компилятор автоматически резолвит `import geometry` в `geometry/geometry.av`.

### Можно ли делать циклические импорты?

Нет. Циклы импортов обнаруживаются и считаются ошибкой.

### Где находятся модули?

Модули ищутся по файловым путям:

- **Стандартные модули** (`std.*`): ищутся в каталоге `std/`
- **Модули приложения**: ищутся относительно корня проекта

Сначала компилятор пробует папочный импорт (`module/A/A.av`), затем — плоский файл (`module/A.av`).

## Функции и методы

### В чём разница между функцией и методом?

Функция — это самостоятельная функция. Метод — это функция, связанная с типом
и объявленная как метод экземпляра или статический метод.

### Можно ли использовать параметры по умолчанию?

Да. Параметры функции могут иметь значения по умолчанию:

```avenir
fun greet(name | string, greeting | string = "Hello") | void {
    // ...
}
```

### Можно ли использовать именованные аргументы?

Да. Вызовы функций поддерживают именованные аргументы:

```avenir
create_point(x = 10, y = 20);
```

### Можно ли возвращать несколько значений?

Не напрямую. Для этого используйте структуру или список:

```avenir
struct Result {
    value | int
    error | error?
}

fun compute() | Result {
    return Result{value = 42, error = none};
}
```

## Управление потоком

### Можно ли использовать `break` внутри `if`?

Нет. `break` можно использовать только внутри циклов.

### Можно ли использовать `continue`?

Пока нет. `continue` ещё не реализован.

### Можно ли использовать `switch`?

Пока нет. Вместо этого используйте цепочки `if-else`.

### Можно ли использовать `defer`?

Пока нет. `defer` ещё не реализован.

## Обработка ошибок

### Как обрабатывать ошибки?

Используйте `try-catch`:

```avenir
try {
} catch (e | error) {
}
```

### Можно ли возвращать ошибки вместо `throw`?

Пока нет. Ошибки должны выбрасываться как исключения.

### Можно ли создавать собственные типы ошибок?

Пока нет. Все ошибки должны иметь тип `error`.

## Производительность

### Быстрый ли Avenir?

VM оптимизирована для выполнения, но скомпилированный нативный код был бы быстрее.

### Чем Avenir отличается от других языков?

Avenir ориентирован на надёжность и ясность. Производительность учитывается, но
не является главной целью.

## Разработка

### Как отлаживать программы на Avenir?

Используйте `print` для отладки. Полноценного отладчика пока нет.

### Как тестировать программы на Avenir?

Пишите тестовые программы и запускайте их. Отдельного фреймворка тестирования пока нет.

### Как профилировать программы на Avenir?

Пока специализированные средства профилирования недоступны.
