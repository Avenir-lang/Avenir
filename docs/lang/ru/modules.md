# Модули

Avenir использует модульную систему для организации кода. Каждый файл — это модуль, определяемый объявлением `pckg`.

## Объявление пакета

Каждый файл должен начинаться с:

```avenir
pckg <name>;
```

Имя пакета может быть простым идентификатором или dotted path:

```avenir
pckg main;
pckg std.net;
pckg app.utils;
```

Имя пакета определяет модуль и используется при импорте.

## Импорты

Модули импортируются через `import`:

```avenir
import <module_path>;
```

Путь импорта — это dotted identifier, совпадающий с именем пакета импортируемого модуля:

```avenir
import std.net;
import std.json as json;
```

### Алиасы импортов

Импорт может иметь алиас:

```avenir
import std.json as json;
```

Если алиас не указан, локальным именем становится последний сегмент пути.

### Использование импортированных модулей

Доступ к импортированным символам осуществляется через точечную нотацию:

```avenir
import std.net as net;

var sock | net.Socket = net.connect("example.com", 80);
```

## Соответствие файла и структуры

**Файл можно импортировать только если он содержит структуру с тем же именем, что и файл.**

### Правило

Если файл содержит структуры, **хотя бы одна структура должна иметь имя файла** без расширения `.av`.

Примеры.

**Корректно:**

```
Файл: Point.av
Содержит: struct Point { x | int; y | int; }
→ Можно импортировать
```

**Некорректно:**

```
Файл: Point.av
Содержит: struct Rectangle { x | int; y | int; }
→ Ошибка компиляции: "file 'Point.av' does not contain struct 'Point'"
```

Файлы без структур можно импортировать свободно: правило применяется только к файлам, где структуры есть.

Если в файле несколько структур, хотя бы одна из них должна совпадать с именем файла.

## Импорты через папки

Если папка `A` содержит файл `A.av`, импорт указывает только на папку:

```
Папка: module/geometry
Файл: module/geometry/geometry.av
Импорт: import module.geometry;
```

Компилятор автоматически резолвит `import module.geometry` в `module/geometry/geometry.av`.

### Порядок разрешения

Для пути вроде `module.A` компилятор пытается:

1. папочный вариант: `module/A/A.av`
2. плоский файл: `module/A.av`

Это правило действует и для `std.*`, и для обычных модулей приложения.

### Вложенные модули

Для вложенных путей применяется та же логика:

```
Папка: app/utils/
Файл: app/utils/utils.av
Пакет: pckg app.utils;
Импорт: import app.utils;
```

## Разрешение модулей

Компилятор разрешает импорты так:

1. Для модулей `std.*`:
   - сначала проверяет `std/<path>/<last-segment>.av`
   - затем проверяет `std/<path>.av`
   - при необходимости использует корень репозитория для каталога `std/`

2. Для остальных модулей:
   - сначала проверяет `<project-root>/<path>/<last-segment>.av`
   - затем проверяет `<project-root>/<path>.av`

Корень проекта определяется как директория entry-файла.

## Многофайловые модули

При папочном импорте модуль может состоять из нескольких файлов. Все `.av` файлы директории объединяются в один модуль.

Пример структуры:

```
std/net/
  net.av
  socket.av
  server.av
```

Если файл содержит структуры, он должен содержать структуру, совпадающую с именем файла. Для вспомогательных файлов можно добавить приватную структуру-заглушку.

## Проверки на этапе компиляции

Компилятор строго проверяет:

- отсутствие файла или папки для импорта
- несовпадение имени структуры и файла
- существование папки без ожидаемого файла

### Сообщения об ошибках

Компилятор старается выдавать понятные сообщения, например:

```
Error: folder "module/geometry" exists but does not contain required file "geometry.av"
Expected: module/geometry/geometry.av
```

## Доступ к модулю

После импорта доступны публичные функции, структуры и интерфейсы модуля:

```avenir
import std.net as net;

fun main() | void {
    var sock | net.Socket = net.connect("example.com", 80);
    sock.close();
}
```

## Видимость модулей

Из других модулей доступны только `pub` функции, структуры и интерфейсы.

## Примечания по алиасам импорта

Если алиас не задан, локальным именем становится последний сегмент пути. Для удобства `std.http.client` и `std.http.server` можно импортировать с `as http`.

## Примеры

### Плоский импорт

```
Файл: utils.av
Пакет: pckg utils;
Импорт: import utils;
```

### Папочный импорт

```
Папка: math/
Файл: math/math.av
Пакет: pckg math;
Импорт: import math;
```

## Структура модуля

Модуль может содержать:

1. объявление пакета
2. импорты
3. структуры
4. интерфейсы
5. функции

## Циклы импортов

Циклы импортов обнаруживаются и возвращаются как ошибки. Модуль не может импортировать сам себя напрямую или косвенно.
