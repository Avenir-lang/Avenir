# Словари (`dict`)

`dict` — встроенный тип словаря в Avenir. Ключи всегда имеют тип `string`, а
значения статически типизированы.

## Синтаксис

Литералы словаря используют фигурные скобки и пары `key: value`:

```avenir
var user | dict<any> = {
    name: "Alex",
    "age": 30,
    isAdmin: true,
};
```

Ключами могут быть:

- идентификаторы (`name`, `isAdmin`)
- строковые литералы (`"age"`, `'role'`)

Завершающая запятая необязательна.

## Типы

Тип словаря записывается как `dict<T>`, где `T` — тип значения:

```avenir
var scores | dict<int> = { alice: 10, bob: 12 };
var meta   | dict<any> = { env: "dev", retries: 3 };
```

Все значения литерала должны быть совместимы с `T`. Смешанные типы значений
требуют явного объединённого типа:

```avenir
var mixed | dict< <int|string> > = { a: 1, b: "two" };
```

## Доступ

Точечный доступ читает ключ:

```avenir
print(user.name);
```

Индексный доступ читает ключ динамически:

```avenir
print(user["age"]);
```

Если при индексном доступе ключ отсутствует, возникает ошибка времени выполнения. Если ключ
может отсутствовать, используйте `dict.get()`.

## Встроенные методы

| Метод | Параметры | Возвращает | Примечания |
| --- | --- | --- | --- |
| `length()` | — | `int` | число записей |
| `keys()` | — | `list<string>` | порядок не гарантирован |
| `values()` | — | `list<T>` | порядок не гарантирован |
| `has(key)` | `string` | `bool` | проверка наличия |
| `get(key)` | `string` | `T?` | `none`, если отсутствует |
| `set(key, value)` | `string`, `T` | `void` | изменяет словарь на месте |
| `remove(key)` | `string` | `bool` | возвращает, существовал ли ключ |

## Словари и структуры

У структур фиксированная схема и именованные поля; словари динамические и
индексируются строками. Используйте структуры для фиксированной формы данных, а
словари — для динамических данных.

## Примечания

- Словари основаны на хеш-таблице; порядок итерации не гарантирован.
- `dict.set` изменяет словарь на месте.
