# Словари (`dict`)

`dict` — встроенный обобщённый тип словаря в Avenir с полной формой
`dict<K, V>`, где `K` — тип ключа, а `V` — тип значения.

Сокращённая форма `dict<V>` эквивалентна `dict<string, V>` (ключи — строки по
умолчанию).

## Синтаксис

Литералы словаря используют фигурные скобки и пары `key: value`:

```avenir
var user | dict<any> = {
    name: "Alex",
    "age": 30,
    isAdmin: true,
};
```

Ключами могут быть:

- идентификаторы (`name`, `isAdmin`)
- строковые литералы (`"age"`, `'role'`)

Завершающая запятая необязательна.

## Типы

Тип словаря может быть записан двумя способами:

**Краткая форма** — `dict<V>` (тип ключа по умолчанию `string`):

```avenir
var scores | dict<int> = { alice: 10, bob: 12 };
var meta   | dict<any> = { env: "dev", retries: 3 };
```

**Полная форма** — `dict<K, V>` (явные типы ключа и значения):

```avenir
var scores | dict<string, int> = { math: 95, science: 88 };
```

Все значения литерала должны быть совместимы с `V`. Смешанные типы значений
требуют явного объединённого типа:

```avenir
var mixed | dict< <int|string> > = { a: 1, b: "two" };
```

## Доступ

Точечный доступ читает ключ:

```avenir
print(user.name);
```

Индексный доступ читает ключ динамически:

```avenir
print(user["age"]);
```

Если при индексном доступе ключ отсутствует, возникает ошибка времени выполнения. Если ключ
может отсутствовать, используйте `dict.get()`.

## Встроенные методы

Для `dict<K, V>`:

| Метод | Параметры | Возвращает | Примечания |
| --- | --- | --- | --- |
| `length()` | — | `int` | число записей |
| `keys()` | — | `list<K>` | порядок не гарантирован |
| `values()` | — | `list<V>` | порядок не гарантирован |
| `has(key)` | `K` | `bool` | проверка наличия |
| `get(key)` | `K` | `V?` | `none`, если отсутствует |
| `set(key, value)` | `K`, `V` | `void` | изменяет словарь на месте |
| `remove(key)` | `K` | `bool` | возвращает, существовал ли ключ |

## Словари и структуры

У структур фиксированная схема и именованные поля; словари динамические и
индексируются строками. Используйте структуры для фиксированной формы данных, а
словари — для динамических данных.

## Примечания

- Словари основаны на хеш-таблице; порядок итерации не гарантирован.
- `dict.set` изменяет словарь на месте.
- В текущей реализации поддерживаются только ключи типа `string`. Параметр типа
  `K` позволяет расширить набор допустимых типов ключей в будущем без изменения
  синтаксиса.
