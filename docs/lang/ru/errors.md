# Обработка ошибок

Avenir использует исключения для обработки ошибок. Исключения представлены типом `error` и возникают в случаях:

- явного `throw`
- ошибок времени выполнения (деление на ноль, неверный доступ и т.д.)
- ошибок встроенных функций

## Тип `error`

Тип `error` представляет исключительные ситуации:

```avenir
var e | error = error("что-то пошло не так");
```

Ошибки создаются через встроенную функцию `error()`. Ошибки времени выполнения и ошибки встроенных функций также автоматически создаются VM как значения `error`.

## Выбрасывание исключений

Исключения выбрасываются через `throw`:

```avenir
throw error("что-то пошло не так");
```

Выражение должно иметь тип `error`.

## Перехват исключений

Исключения перехватываются через `try-catch`:

```avenir
try {
    throw error("произошла ошибка");
} catch (e | error) {
    print(errorMessage(e));
}
```

Ошибки времени выполнения и ошибки встроенных функций тоже можно перехватывать:

```avenir
try {
    var n | int = toInt("abc");
    var x | int = 10 / 0;
    print(n + x);
} catch (e | error) {
    print(errorMessage(e));
}
```

Переменная в `catch` должна иметь тип `error`.

## Сообщения об ошибках

Текст ошибки извлекается с помощью `errorMessage()`:

```avenir
var msg | string = errorMessage(e);
```

## Распространение исключений

Если исключение не перехвачено, оно поднимается вверх по стеку вызовов:

```avenir
fun f() | void {
    throw error("ошибка в f");
}

fun g() | void {
    f();
}

fun main() | void {
    try {
        g();
    } catch (e | error) {
    }
}
```

Если исключение доходит до верхнего уровня и не перехватывается, программа завершается с ошибкой.

## Необработанные исключения

Необработанные исключения приводят к завершению программы:

```avenir
fun main() | void {
    throw error("необработанная ошибка");
}
```

## Шаблоны обработки ошибок

### Проброс ошибок

Функция может пробросить ошибку дальше, просто не перехватывая её:

```avenir
fun may_fail() | void {
    if (condition) {
        throw error("сбой");
    }
}
```

### Обработка ошибок

Функция может обработать ошибку через `try-catch`:

```avenir
fun safe_operation() | void {
    try {
        may_fail();
    } catch (e | error) {
        print(errorMessage(e));
    }
}
```

### Возврат ошибок

Сейчас Avenir использует прежде всего исключения. Функции могут возвращать `error`, но исключения всё равно выбрасываются через `throw` или возникают из ошибок времени выполнения и встроенных функций.

## Ограничения типа `error`

`error` — встроенный тип с сообщением и расширяемыми метаданными. Пользовательские типы ошибок пока не поддерживаются.

## Практические рекомендации

- корректно обрабатывайте ошибки
- делайте сообщения понятными
- используйте `try-catch` там, где действительно можете обработать исключение
- документируйте функции, которые могут бросать ошибки
