# Методы

Методы — это функции, связанные с типом. Avenir поддерживает методы экземпляра и статические методы.

## Методы экземпляра

Метод экземпляра — это функция с параметром-получателем.

### Объявление

Методы экземпляра объявляются через синтаксис получателя:

```avenir
fun (self | Point).sum() | int {
    return self.x + self.y;
}
```

Синтаксис получателя: `(name | Type).methodName`.

### Вызов методов экземпляра

Методы вызываются через точечную нотацию:

```avenir
var p | Point = Point{x = 3, y = 4};
var s | int = p.sum();
```

### Доступ к приватным полям

Методы экземпляра могут обращаться к приватным полям получателя.

```avenir
struct Point {
    x | int
    y | int
}

fun (self | Point).sum() | int {
    return self.x + self.y;
}
```

### Изменяющие методы

Методы экземпляра могут изменять изменяемые поля.

```avenir
mut struct Point {
    x | int
    y | int
}

fun (self | Point).move(dx | int, dy | int) | void {
    self.x = self.x + dx;
    self.y = self.y + dy;
}
```

## Статические методы

Статические методы связаны с типом, но не имеют экземпляра-получателя.

### Объявление

Статические методы объявляются через имя типа:

```avenir
fun Point.origin() | Point {
    return Point{x = 0, y = 0};
}
```

Синтаксис: `Type.methodName`.

### Вызов статических методов

Статические методы вызываются через тип.

```avenir
var origin | Point = Point.origin();
```

У статических методов нет доступа к полям конкретного экземпляра.

## Видимость методов

Для методов действуют те же правила видимости, что и для функций:

- **Публичные методы**: с `pub`, доступны из других модулей
- **Приватные методы**: без `pub`, доступны только внутри модуля

```avenir
pub struct Point {
    x | int
    y | int
}

pub fun (self | Point).distance() | float {
}

fun (self | Point).internal() | void {
}
```

## Перегрузка методов

Avenir не поддерживает перегрузку методов. Имя метода должно быть уникальным внутри типа.

## Методы и функции

Методы — это по сути функции с получателем.

Следующее концептуально эквивалентно:

```avenir
fun (self | Point).sum() | int {
    return self.x + self.y;
}

fun point_sum(p | Point) | int {
    return p.x + p.y;
}
```

Методы дают более естественный синтаксис для объектно-ориентированного стиля.
