# Лексер

Этот документ описывает лексический анализ в Avenir: реализацию в
`internal/lexer/lexer.go` и определения токенов в `internal/token/token.go`.

## Обзор

Лексер преобразует исходный текст в поток токенов с позициями строка/столбец.
Он работает на рунах (UTF-8) и отслеживает текущий символ, строку и столбец
для точной диагностики ошибок.

## Категории токенов

Лексер распознаёт:

- **Идентификаторы** и **ключевые слова**
- **Числа**: целые и с плавающей точкой
- **Строки**: в одинарных и двойных кавычках
- **Байтовые литералы**: `b"..."` (только двойные кавычки)
- **Операторы**: `+`, `-`, `*`, `/`, `%`, `==`, `!=`, `<`, `<=`, `>`, `>=`, `&&`, `||`, `!`
- **Символы**: `(` `)` `{` `}` `[` `]` `.` `,` `;` `:` `|` `?`
- **Маркеры интерполяции**: `${`, `}`, а также токены частей строки
- Токены **EOF** и **Illegal**

Канонический список токенов: `internal/token/token.go`.

## Пробелы и комментарии

- Пробельные символы Unicode пропускаются.
- Однострочные комментарии начинаются с `//`.
- Блочные комментарии ограничиваются `/* ... */` и могут быть многострочными.

## Числа

Лексер выделяет:

- **Int**: только цифры, например `42`
- **Float**: цифры с `.` или экспонентой `e/E`, например `3.14`, `1e3`

Тип токена (`Int` или `Float`) определяется по наличию `.` или `e/E`.

## Строки

Строки ограничиваются `"` или `'`. Обе формы имеют одинаковую семантику.

### Экранирование

Экранирующие последовательности обрабатываются лексером (в AST уже декодированная строка):

| Escape | Значение |
| --- | --- |
| `\\` | обратный слеш |
| `\"` | двойная кавычка |
| `\'` | одинарная кавычка |
| `\n` | перевод строки |
| `\t` | табуляция |
| `\r` | возврат каретки |
| `\0` | нулевой байт |
| `\uXXXX` | кодовая точка Unicode |
| `\xNN` | байтовое значение |

Некорректные escape-последовательности и незакрытые строки фиксируются как
ошибки лексера и порождают `Illegal` токены.

### Интерполяция

Интерполированные строки используют `${ ... }`. Лексер порождает последовательность:

- `StringPart` для текстовых сегментов
- `InterpStart` для `${`
- токены выражения внутри интерполяции
- `InterpEnd` для `}`
- `StringEnd` при завершении интерполированной строки

Неинтерполированные строки дают один токен `String`.

Пример:

```avenir
"hello ${name}"
```

Поток токенов (упрощённо):

```
StringPart("hello ")
InterpStart
Ident(name)
InterpEnd
StringEnd
```

## Байтовые литералы

Байтовые литералы используют форму `b"..."` и поддерживают те же escape-последовательности.
Интерполяция в байтовых литералах не поддерживается.

## Отчёты об ошибках

Лексер накапливает ошибки как строки с позициями (строка/столбец).
Потребитель может получить их через `lexer.Errors()`.
