# Память и производительность

Этот документ описывает поведение памяти в runtime и ключевые аспекты производительности.

## Семантика значений

Большинство значений неизменяемы на уровне языка:

- `string` и `bytes` неизменяемы (операции возвращают новые значения)
- методы `list` возвращают новые списки (без мутаций in-place)
- `dict.set` изменяет словарь на месте

Значения структур копируются при присваивании (value semantics), но содержат
ссылки на внутреннее хранилище list/dict/bytes (поверхностная копия).

## Списки

Списки реализованы через Go slices. Операции, создающие списки, выделяют новые
срезы и копируют элементы. При присваивании копируется заголовок среза, поэтому
несколько переменных могут ссылаться на один underlying slice.

## Словари

Dict основаны на Go map. `dict.set` и `dict.remove` мутируют map на месте,
поэтому совместное использование dict разделяет состояние.

## Строки

Строки — Go strings. Интерполяция lowering-ится в `OpStringify` и
`OpConcatString`. Для конкатенации VM использует `strings.Builder` для
уменьшения аллокаций.

## Bytes

Bytes представлены как `[]byte`. Методы `append`, `concat`, `slice` выделяют
новые байтовые срезы и копируют данные.

## Структуры

Поля структур хранятся в срезе внутри `value.Value`.
Присваивание поля обновляет значение структуры на месте (для mutable-полей).

## Ошибки как значения

Ошибки — обычные значения (`value.KindError`) и распространяются VM при runtime-сбоях.

## Примечания по оптимизации

Текущая IR/VM-реализация ориентирована на простоту и читаемость, а не на
агрессивные оптимизации. Классические оптимизации (constant folding,
common subexpression elimination) пока не реализованы.
