# Ошибки и диагностика

Этот документ описывает обработку ошибок в компиляторе и runtime.

## Ошибки времени компиляции

### Ошибки лексера

Лексер фиксирует неверные токены (незакрытые строки, невалидные escape-последовательности и т.д.)
с позициями строка/столбец. Ошибки доступны через `lexer.Errors()`.

### Ошибки парсера

Парсер накапливает синтаксические ошибки через `p.errorf(...)`.
Разбор продолжается, чтобы показать несколько ошибок за один запуск.

### Ошибки проверки типов

Type checker возвращает элементы `types.Error` с позициями и сообщениями.
Типичные случаи:

- несовпадение типов
- неизвестные имена
- некорректные присваивания
- ошибки соответствия интерфейсам

## Ошибки времени выполнения

В runtime все ошибки представлены значениями типа `error` и могут быть выброшены:

- деление на ноль
- выход за границы индекса
- неверное использование builtin-функций
- ошибки I/O или сети

## Семантика try/catch

Компилятор генерирует:

- `OpBeginTry` с адресом обработчика
- `OpEndTry` для удаления обработчика
- `OpThrow` для выброса значения

VM хранит обработчики в стеке. При ошибке VM:

1. разматывает стек до ближайшего обработчика
2. восстанавливает глубину стека
3. кладёт значение ошибки
4. прыгает на IP обработчика

Если обработчика нет, ошибка поднимается до входной точки VM и завершает выполнение.

### Типизированные catch-клаузы

При наличии нескольких типизированных catch-клауз компилятор генерирует цепочку
проверок типов в точке входа обработчика:

1. `OpIsStructType A` (A = индекс типа структуры) — peek TOS, push bool
2. `OpJumpIfFalse` к следующей клаузе
3. При совпадении: сохранение значения в локальную переменную, тело клаузы, `OpJump` в конец
4. Финальный `OpThrow` перебрасывает необработанные ошибки

`catch (e | error)` — fallback-клауза, перехватывающая любую ошибку.

### Декларация throws

Функции могут объявлять типы выбрасываемых ошибок через `!` после типа возврата:

```avenir
fun f() | void ! FileNotFound, PermissionDenied { ... }
```

Тип `Func` содержит поле `Throws []Type`.

## Примечания

- `throw` принимает значения типа `error` и значения структур.
- Значения структур, выброшенные как ошибки, передаются в catch-обработчики
  без оборачивания.
- `errorMessage(e)` извлекает сообщение для пользователя из значений типа `error`.
