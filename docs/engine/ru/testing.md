# Тестирование движка

Этот документ описывает, как тестировать компоненты компилятора и runtime.

## Структура тестов

Тесты организованы по пакетам, например:

- `internal/lexer/*_test.go`
- `internal/parser/*_test.go`
- `internal/types/*_test.go`
- `internal/ir/*_test.go`
- `internal/vm/*_test.go`
- `internal/runtime/builtins/*_test.go`

## Запуск тестов

Из корня репозитория:

```bash
go test ./...
```

## Тесты лексера и парсера

Типичный сценарий:

1. Лексинг исходника в токены
2. Парсинг в AST
3. Проверка структуры узлов или сообщений об ошибках

## Тесты type checker

Используйте `types.CheckWorldWithBindings` для проверки типов `modules.World`
или проверяйте отдельную программу напрямую. Проверяйте ошибки и bindings.

## Тесты IR и VM

IR-тесты проверяют, что компилятор генерирует ожидаемые опкоды
(или хотя бы ожидаемый результат исполнения).
VM-тесты обычно компилируют небольшую программу, вызывают `vm.RunMain()`
и проверяют результат.

## Пример end-to-end теста (концептуально)

```go
src := `
pckg main;
fun main() | void { print("ok"); }
`

// lex/parse
// type check
// compile to IR
// run VM
```

## Советы

- Используйте маленькие и целевые тестовые программы.
- По возможности проверяйте возвращаемые ошибки, а не строки целиком.
- Добавляйте тесты runtime-ошибок (например, выход за пределы индекса).
