# Типы и проверка типов

Этот документ описывает статическую систему типов и type checker в
`internal/types`.

## Обзор

Проверка типов явная и статическая: все переменные и параметры должны иметь
объявленные типы. Checker валидирует структуру программы, разрешает имена и
формирует binding-метаданные для IR-компилятора.

Ключевые файлы:

- `internal/types/types.go`
- `internal/types/checker.go`

## Виды типов

### Базовые типы

`int`, `float`, `string`, `bool`, `bytes`, `void`, `any`, `error`.

### Списки

`list<T>` поддерживает несколько типов элементов:

- `list<int>`
- `list<int, string>`

Внутри `List.ElementTypes` хранится набор допустимых типов элементов.

### Словари

`dict<T>` отображает строковые ключи в значения типа `T`.

### Опциональные типы

`T?` — опциональный тип. `none` — единственное null-подобное значение,
`some(expr)` оборачивает значение в optional.

### Union-типы

`<A|B|C>` позволяет хранить значение одного из вариантов.

### Структуры и интерфейсы

- Структуры — номинальные типы.
- Интерфейсы — структурные типы.

### Типы функций

`fun(T1, T2) | R` для first-class функций.

### Future-типы (async)

Checker моделирует async-возвраты через `Future<T>`:

- `async fun ... | T` регистрируется в scope как `fun(...) | Future<T>`.
- Внутри async-функции `return expr;` проверяется против `T`, а не `Future<T>`.
- `await expr` требует тип `Future<X>` и возвращает `X`.

Если `await` применяется к не-future выражению, checker сообщает ошибку.

### Дженерики

Поддерживаются явные дженерики для структур и функций:

- `struct Box<T> { ... }`
- `fun identity<T>(x | T) | T { ... }`
- использование: `Box<int>`, `identity<int>(10)`

Вывод generic-аргументов пока не реализован.

## Разрешение имён

В scope добавляются:

- builtin-функции
- локальные объявления
- символы импортированных модулей (алиас + полный путь)

## Поток работы checker-а

1. Объявление builtins.
2. Загрузка модулей.
3. Объявление типов.
4. Объявление функций и методов.
5. Проверка всех выражений и операторов.
6. Формирование `Bindings` для IR.

Для дженериков добавляются шаги мономорфизации (регистрация шаблонов,
инстанцирование, генерация имён вида `Box$int`).

## Правила присваивания

- `any` принимает любой тип.
- `T` присваивается в `T?`.
- `none` присваивается любому optional-типу.
- union принимает совместимые варианты.
- list/dict проверяются по совместимости элементов/значений.
- структуры требуют совпадения имени.
- интерфейсы требуют полного совпадения сигнатур методов.

## Builtins и методы

Builtins регистрируются через `builtins.Meta` и добавляются в scope как функции.
Builtin-методы разрешаются по типу receiver-а.

## Выходные данные Bindings

`Bindings` содержит:

- `Idents`, `Members`, `ExprTypes`
- `MonomorphizedStructs`
- `MonomorphizedFuncs`

## Ошибки

Ошибки типов возвращаются как срез `types.Error` с позициями в исходнике.
