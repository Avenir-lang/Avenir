# Пам'ять і продуктивність

Цей документ описує поведінку пам'яті в runtime та ключові аспекти продуктивності.

## Семантика значень

Більшість значень незмінні на рівні мови:

- `string` і `bytes` незмінні (операції повертають нові значення)
- методи `list` повертають нові списки (без мутацій in-place)
- `dict.set` змінює словник на місці

Значення структур копіюються під час присвоєння (value semantics), але містять
посилання на внутрішнє сховище list/dict/bytes (поверхнева копія).

## Списки

Списки реалізовані через Go slices. Операції, що створюють списки, виділяють
нові зрізи і копіюють елементи. При присвоєнні копіюється заголовок зрізу,
тому кілька змінних можуть посилатися на один underlying slice.

## Словники

Dict базуються на Go map. `dict.set` і `dict.remove` мутують map на місці,
тому спільне використання dict розділяє стан.

## Рядки

Рядки — Go strings. Інтерполяція lowering-иться в `OpStringify` і
`OpConcatString`. Для конкатенації VM використовує `strings.Builder` для
зменшення алокацій.

## Bytes

Bytes представлені як `[]byte`. Методи `append`, `concat`, `slice` виділяють
нові байтові зрізи і копіюють дані.

## Структури

Поля структур зберігаються у зрізі всередині `value.Value`.
Присвоєння поля оновлює значення структури на місці (для mutable-полів).

## Помилки як значення

Помилки — звичайні значення (`value.KindError`) і поширюються VM при runtime-збоях.

## Примітки щодо оптимізації

Поточна IR/VM-реалізація орієнтована на простоту й читабельність, а не на
агресивні оптимізації. Класичні оптимізації (constant folding,
common subexpression elimination) поки не реалізовані.
