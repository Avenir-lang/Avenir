# Типи і перевірка типів

Цей документ описує статичну систему типів і type checker у
`internal/types`.

## Огляд

Перевірка типів статична. Змінні можуть використовувати явні анотації типів
або локальне виведення типів (`var name = expr;`). Виклики узагальнених функцій
підтримують виведення аргументів типів з аргументів виклику. Checker валідує структуру програми, резолвить імена і
формує binding-метадані для IR-компілятора.

Ключові файли:

- `internal/types/types.go`
- `internal/types/checker.go`

## Види типів

### Базові типи

`int`, `float`, `string`, `bool`, `bytes`, `void`, `any`, `error`.

### Списки

`list<T>` підтримує кілька типів елементів:

- `list<int>`
- `list<int, string>`

Всередині `List.ElementTypes` зберігається набір допустимих типів елементів.

### Словники

`dict<K, V>` відображає ключі типу `K` у значення типу `V`. Скорочена форма
`dict<V>` за замовчуванням використовує `K = string`.

Всередині `Dict.KeyType` дорівнює `nil` при використанні скороченої форми;
метод-хелпер `keyType()` повертає `String` у цьому випадку.

### Опціональні типи

`T?` — опціональний тип. `none` — єдине null-подібне значення,
`some(expr)` обгортає значення в optional.

### Union-типи

`<A|B|C>` дозволяє зберігати значення одного з варіантів.

### Структури та інтерфейси

- Структури — номінальні типи.
- Інтерфейси — структурні типи.

### Типи функцій

`fun(T1, T2) | R` для first-class функцій.

Функції можуть оголошувати типи помилок, що викидаються: `Func.Throws []Type`.
Type checker валідує вирази `throw` проти оголошених throws і відстежує їх
через поле `currentThrows`.

### Future-типи (async)

Checker моделює async-повернення через `Future<T>`:

- `async fun ... | T` реєструється в scope як `fun(...) | Future<T>`.
- Усередині async-функції `return expr;` перевіряється проти `T`, а не `Future<T>`.
- `await expr` вимагає тип `Future<X>` і повертає `X`.

Якщо `await` застосовано до не-future виразу, checker повертає помилку.

### Дженеріки

Підтримуються явні дженеріки для структур і функцій:

- `struct Box<T> { ... }`
- `fun identity<T>(x | T) | T { ... }`
- використання: `Box<int>`, `identity<int>(10)`

Виведення generic-аргументів підтримується при виклику узагальнених функцій.

## Резолв імен

У scope додаються:

- builtin-функції
- локальні оголошення
- символи імпортованих модулів (аліас + повний шлях)

## Потік роботи checker-а

1. Оголошення builtins.
2. Завантаження модулів.
3. Оголошення типів.
4. Оголошення функцій і методів.
5. Перевірка всіх виразів і операторів.
6. Формування `Bindings` для IR.

Для дженеріків додаються кроки мономорфізації (реєстрація шаблонів,
інстанціювання, генерація імен виду `Box$int`).

## Правила присвоєння

- `any` приймає будь-який тип.
- `T` присвоюється в `T?`.
- `none` присвоюється будь-якому optional-типу.
- union приймає сумісні варіанти.
- list перевіряється за сумісністю типів елементів.
- dict перевіряється за сумісністю типу ключа і типу значення.
- структури вимагають збігу імені.
- інтерфейси вимагають повного збігу сигнатур методів.

## Builtins і методи

Builtins реєструються через `builtins.Meta` і додаються в scope як функції.
Builtin-методи резолвляться за типом receiver-а.

## Вихідні дані Bindings

`Bindings` містить:

- `Idents`, `Members`, `ExprTypes`
- `MonomorphizedStructs`
- `MonomorphizedFuncs`

## Помилки

Помилки типів повертаються як зріз `types.Error` з позиціями у вихіднику.
