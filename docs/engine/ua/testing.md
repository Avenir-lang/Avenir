# Тестування рушія

Цей документ описує, як тестувати компоненти компілятора і runtime.

## Структура тестів

Тести організовані за пакетами, наприклад:

- `internal/lexer/*_test.go`
- `internal/parser/*_test.go`
- `internal/types/*_test.go`
- `internal/ir/*_test.go`
- `internal/vm/*_test.go`
- `internal/runtime/builtins/*_test.go`

## Запуск тестів

З кореня репозиторію:

```bash
go test ./...
```

## Тести лексера і парсера

Типовий сценарій:

1. Лексинг вихідного коду в токени
2. Парсинг в AST
3. Перевірка структури вузлів або повідомлень про помилки

## Тести type checker

Використовуйте `types.CheckWorldWithBindings` для перевірки типів `modules.World`
або перевіряйте окрему програму напряму. Перевіряйте помилки і bindings.

## Тести IR і VM

IR-тести перевіряють, що компілятор генерує очікувані опкоди
(або хоча б очікуваний результат виконання).
VM-тести зазвичай компілюють невелику програму, викликають `vm.RunMain()`
і перевіряють результат.

## Приклад end-to-end тесту (концептуально)

```go
src := `
pckg main;
fun main() | void { print("ok"); }
`

// lex/parse
// type check
// compile to IR
// run VM
```

## Поради

- Використовуйте маленькі й цільові тестові програми.
- За можливості перевіряйте повернені помилки, а не рядки цілком.
- Додавайте тести runtime-помилок (наприклад, вихід за межі індексу).
