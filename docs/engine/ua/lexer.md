# Лексер

Цей документ описує лексичний аналіз у Avenir: реалізацію в
`internal/lexer/lexer.go` і визначення токенів у `internal/token/token.go`.

## Огляд

Лексер перетворює вихідний текст на потік токенів з позиціями рядок/стовпчик.
Він працює на рунах (UTF-8) і відстежує поточний символ, рядок і стовпчик
для точної діагностики помилок.

## Категорії токенів

Лексер розпізнає:

- **Ідентифікатори** та **ключові слова**
- **Числа**: цілі та з плаваючою крапкою
- **Рядки**: в одинарних і подвійних лапках
- **Байтові літерали**: `b"..."` (лише подвійні лапки)
- **Оператори**: `+`, `-`, `*`, `/`, `%`, `==`, `!=`, `<`, `<=`, `>`, `>=`, `&&`, `||`, `!`
- **Символи**: `(` `)` `{` `}` `[` `]` `.` `,` `;` `:` `|` `?`
- **Маркери інтерполяції**: `${`, `}`, а також токени частин рядка
- Токени **EOF** та **Illegal**

Канонічний список токенів: `internal/token/token.go`.

## Пробіли та коментарі

- Усі Unicode-пробіли пропускаються.
- Однорядкові коментарі починаються з `//`.
- Блокові коментарі обмежуються `/* ... */` і можуть бути багаторядковими.

## Числа

Лексер виділяє:

- **Int**: лише цифри, наприклад `42`
- **Float**: цифри з `.` або експонентою `e/E`, наприклад `3.14`, `1e3`

Тип токена (`Int` або `Float`) визначається за наявністю `.` або `e/E`.

## Рядки

Рядки обмежуються `"` або `'`. Обидві форми мають однакову семантику.

### Екранування

Escape-послідовності обробляються лексером (в AST потрапляє вже декодований рядок):

| Escape | Значення |
| --- | --- |
| `\\` | зворотний слеш |
| `\"` | подвійна лапка |
| `\'` | одинарна лапка |
| `\n` | новий рядок |
| `\t` | табуляція |
| `\r` | повернення каретки |
| `\0` | нульовий байт |
| `\uXXXX` | кодова точка Unicode |
| `\xNN` | байтове значення |

Некоректні escape-послідовності та незакриті рядки фіксуються як помилки
лексера і породжують токени `Illegal`.

### Інтерполяція

Інтерпольовані рядки використовують `${ ... }`. Лексер породжує послідовність:

- `StringPart` для текстових сегментів
- `InterpStart` для `${`
- токени виразу в інтерполяції
- `InterpEnd` для `}`
- `StringEnd` при завершенні інтерпольованого рядка

Неінтерпольовані рядки дають один токен `String`.

Приклад:

```avenir
"hello ${name}"
```

Потік токенів (спрощено):

```
StringPart("hello ")
InterpStart
Ident(name)
InterpEnd
StringEnd
```

## Байтові літерали

Байтові літерали використовують форму `b"..."` і підтримують ті ж escape-послідовності.
Інтерполяція в байтових літералах не підтримується.

## Звіти про помилки

Лексер накопичує помилки як рядки з позиціями (рядок/стовпчик).
Споживач може отримати їх через `lexer.Errors()`.
