# Помилки та діагностика

Цей документ описує обробку помилок у компіляторі та runtime.

## Помилки часу компіляції

### Помилки лексера

Лексер фіксує невалідні токени (незакриті рядки, некоректні escape-послідовності тощо)
з позиціями рядок/стовпчик. Помилки доступні через `lexer.Errors()`.

### Помилки парсера

Парсер накопичує синтаксичні помилки через `p.errorf(...)`.
Розбір продовжується, щоб показати кілька помилок за один запуск.

### Помилки перевірки типів

Type checker повертає елементи `types.Error` з позиціями та повідомленнями.
Типові випадки:

- невідповідність типів
- невідомі імена
- некоректні присвоєння
- помилки відповідності інтерфейсам

## Помилки часу виконання

У runtime всі помилки представлені значеннями типу `error` і можуть бути викинуті:

- ділення на нуль
- вихід за межі індексу
- неправильне використання builtin-функцій
- помилки I/O або мережі

## Семантика try/catch

Компілятор генерує:

- `OpBeginTry` з адресою обробника
- `OpEndTry` для видалення обробника
- `OpThrow` для викидання значення

VM зберігає обробники у стеку. При помилці VM:

1. розмотує стек до найближчого обробника
2. відновлює глибину стека
3. кладе значення помилки
4. переходить на IP обробника

Якщо обробника немає, помилка підіймається до точки входу VM і завершує виконання.

## Примітки

- `throw` очікує значення типу `error`; інші значення обгортаються в помилку.
- `errorMessage(e)` витягує повідомлення для користувача.
